명령어 두 개면 충분합니다. 여기서 AI 에이전트 팀이 기능을 병렬로 작업하는 단계까지 바로 갑니다.

1. **태스크 실행**

   만들고 싶은 것을 설명하세요:

   ```
   /relay:relay "장바구니 기능 추가해줘"
   ```

   첫 실행 시 relay가 코드베이스를 분석해 프로젝트에 맞는 에이전트 풀(`.relay/agents.pool.yml`)을 자동 생성합니다 — 기술 스택, 프레임워크, 테스트 러너 등을 감지합니다. 이 파일은 언제든 커스터마이즈할 수 있습니다.

   모든 에이전트가 동시에 시작합니다. 각자 태스크를 클레임하고, MCP 도구로 소통하며,
   P2P로 협업합니다 — 오케스트레이터 병목 없음. 세션 시작 시 출력되는 대시보드 URL에서 실시간으로 확인하세요.

   <Aside type="tip">
   첫 세션 후 `.relay/memory/`를 git에 커밋하세요. 팀 전원이 동일한 에이전트 컨텍스트를 공유합니다.
   </Aside>

2. **대시보드 확인 (선택사항)**

   에이전트가 실행되는 동안 실시간 대시보드를 여세요:

   ```
   http://localhost:3456   # 기본값 — 포트가 이미 사용 중이면 3457~3465 중 자동 선택됨
   ```

   헤더의 세션 진행 위젯, 태스크 완료 바가 있는 Agent Arena, 키보드 내비게이션을 지원하는 Activity Feed, 의존성 표시가 있는 Task Board — 작업이 진행되면서 모두 실시간 업데이트됩니다.

3. **단일 에이전트 호출 (필요할 때)**

   전체 팀 없이 특정 관점 하나만 필요할 때:

   ```
   /relay:agent fe "CartItem 컴포넌트 리팩터링해줘"
   /relay:agent be "사용자 인증 API 보안 검토해줘"
   ```

## 실제 세션은 이렇게 진행됩니다

```
사용자: /relay:relay "로그인 기능 구현해줘"

[PM]       → 로그인을 5개 태스크로 분해: UI, API, 화면 스펙, 이벤트 설계, 테스트
             → Designer, DA, FE, BE에 할당

[Designer] → 로그인 화면 스펙 작성 → post_artifact("login-screen-spec")
[DA]       → login_success / login_fail 이벤트 설계 → post_artifact("login-events")

[FE]       → Designer 스펙 읽기 → 로그인 UI 빌드 → post_artifact("fe-pr")
[BE]       → API 계약 설계 → FE와 공유 → 구현 → post_artifact("be-pr")

[FE2]      → fe-pr 읽기 → submit_review (코멘트와 함께 승인)
[BE2]      → be-pr 읽기 → submit_review (승인)

[QA]       → 테스트 시나리오 작성 → 버그 티켓 생성 → 개발팀이 수정

[Deployer] → QA 승인 → 배포 → 팀에 알림

사용자: 로그인 기능이 배포되었습니다.
```

전체 흐름이 이벤트 드리븐입니다. 에이전트들은 서로의 메시지와 아티팩트에 반응합니다 — 줄을 서서 기다리지 않습니다.
