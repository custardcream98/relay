Run this when you first bring relay into a project. Every agent in your pool scans the codebase simultaneously, each from their own perspective, and writes what they learned to `.relay/memory/`. Every session that follows starts with that shared context.

## When to run

- **Once per project** — the first time you use relay
- **Re-run** — when the codebase structure changes significantly

If `.relay/memory/` doesn't exist and you run `/relay:relay`, relay will prompt you to run this first.

## What happens

### If no agent pool is configured

If `.relay/agents.pool.yml` doesn't exist yet, relay analyzes the project and suggests a team for your domain:

```
[1] Create .relay/agents.pool.yml with this pool (recommended)
[2] Use the example pool (agents.pool.example.yml)
[3] I'll create agents.pool.yml myself
```

Once the pool file is created, the scan proceeds automatically.

### Parallel scan

**All agents in the pool spawn at the same time** and explore the codebase from their role's perspective — the FE agent looks at component structure, the BE agent at API patterns, the PM at overall architecture.

When each agent finishes, it writes key findings to its own memory file. One agent then synthesizes everything into a unified `project.md` summary.

## Output

```
.relay/
└── memory/
    ├── project.md        ← consolidated summary (injected into every session)
    └── agents/
        ├── pm.md
        ├── fe.md
        ├── be.md
        └── ...           ← per-agent personal memory
```

<Aside type="tip">
Commit `.relay/memory/` to git. Everyone on your team shares the same agent context.
</Aside>

## Next step

Once init completes, start working immediately:

```
/relay:relay "add a shopping cart"
```
