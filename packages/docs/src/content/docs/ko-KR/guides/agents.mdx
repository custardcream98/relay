relay의 에이전트 페르소나는 에이전트 풀에 정의됩니다. `/relay:relay` 세션마다 대화형 팀 구성 과정을 통해 태스크에 최적화된 팀을 풀에서 선발합니다.

## 파일 구조

```
your-project/
└── .relay/
    └── agents.pool.yml   # 에이전트 풀 (이 파일을 수정하세요)
```

시작하려면 relay에 포함된 `agents.pool.example.yml`을 `.relay/agents.pool.yml`로 복사하세요.

## agents.pool.yml 기본 구조

```yaml
agents:
  fe:
    name: "Frontend Engineer"
    emoji: "🎨"
    description: "React, TypeScript, UI 구현"
    tags: [frontend, react, typescript]
    tools:
      - get_messages
      - get_my_tasks
      - claim_task
      - update_task
      - post_artifact
      - send_message
      - request_review
    systemPrompt: |
      You are a Frontend Engineer specialising in React and TypeScript.
      ...
```

## extends

다른 에이전트의 설정을 상속해 확장합니다. 같은 역할을 여러 인스턴스로 실행할 때 유용합니다:

```yaml
agents:
  fe:
    name: "Frontend Engineer"
    emoji: "🎨"
    tags: [frontend]
    tools: [claim_task, get_my_tasks, update_task, post_artifact, send_message]
    systemPrompt: |
      You are a Frontend Engineer specialising in React and TypeScript.
      ...

  fe2:
    extends: fe          # fe 페르소나를 상속
    name: "Frontend Engineer (Reviewer)"
    emoji: "🔍"
```

## tags 필드

`tags`는 `/relay:relay`가 스마트 팀 제안 매칭에 사용합니다. 에이전트의 도메인과 기술을 설명하는 태그를 추가하세요:

```yaml
agents:
  security:
    name: "Security Engineer"
    emoji: "🔐"
    tags: [security, owasp, vulnerability]
    description: "Security vulnerability analysis and remediation"
    tools:
      - get_messages
      - get_my_tasks
      - update_task
      - get_artifact
      - post_artifact
      - send_message
    systemPrompt: |
      You are a security expert.
      Review code against the OWASP Top 10 and
      identify vulnerabilities with recommended fixes.
```

## tools 필드

에이전트가 호출할 수 있는 MCP 도구 목록을 지정합니다:

```yaml
tools:
  - send_message
  - get_messages
  - create_task
  - update_task
  - get_my_tasks
  - post_artifact
  - get_artifact
  - request_review
  - submit_review
  - read_memory
  - write_memory
  - append_memory
```

`tools` 배열에 없는 MCP 도구는 에이전트가 호출할 수 없습니다.
각 역할에 필요한 최소한의 도구만 부여하는 것을 권장합니다.
