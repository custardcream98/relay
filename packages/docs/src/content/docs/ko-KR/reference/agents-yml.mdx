`.relay/agents.pool.yml`은 동적 팀 구성에 사용되는 에이전트 풀을 정의합니다.
`/relay:relay` 세션은 매번 이 풀에서 태스크에 맞는 팀을 선택합니다.

## 최상위 구조

```yaml
language: string   # 선택 사항 — 전체 에이전트 기본 언어
agents:
  {agentId}:
    # 에이전트 설정
    ...
```

## 에이전트 스키마

```yaml
agents:
  {agentId}:
    name: string           # 표시 이름 (예: "Frontend Engineer")
    emoji: string          # 이모지 아이콘 (예: "💻")
    description: string    # 짧은 역할 설명 (팀 선택 화면에 표시)
    tags: [string]         # 선택 사항 — 팀 자동 추천에 사용
    disabled: boolean      # true이면 풀에서 제외 (기본값: false)
    extends: string        # 상속할 에이전트 ID
    tools:                 # 허용된 MCP 툴 목록
      - tool_name
    systemPrompt: string   # 에이전트 시스템 프롬프트 (멀티라인)
    language: string       # 선택 사항 — 이 에이전트의 언어 개별 설정
```

## extends 패턴

`extends`를 사용하면 동일한 풀 페르소나의 여러 인스턴스를 다른 ID로 생성할 수 있습니다.
`/relay:relay`가 세션 파일 작성 시 `fe`, `fe2`, `fe3` 등을 구성할 때 이 패턴을 사용합니다:

```yaml
# .relay/session-agents-2026-03-14-001-a3f7.yml
agents:
  fe:
    extends: fe   # 풀의 systemPrompt + tools 상속
    name: "Frontend Engineer 1"
  fe2:
    extends: fe
    name: "Frontend Engineer 2"
```

## tags 필드

`tags`는 `/relay:relay`가 팀을 자동 추천할 때 사용됩니다.
태스크 유형에 맞는 태그를 지정해 두면 스킬이 적합한 에이전트를 제안합니다:

```yaml
agents:
  be:
    tags: [backend, api, database, node, typescript]
  researcher:
    tags: [research, analysis, writing, synthesis]
```


## 파일 위치

풀 파일은 우선순위 순으로 검색됩니다:
1. `.relay/agents.pool.yml` — 프로젝트 레벨 (우선순위 높음, git 추적)
2. `agents.pool.yml` — 프로젝트 루트 폴백

파일이 없으면 relay가 다음 오류를 반환합니다:
```
No agent pool configured. Create .relay/agents.pool.yml (see agents.pool.example.yml).
```
