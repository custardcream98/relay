relay의 에이전트 메모리는 두 레이어로 나뉩니다: **세션 메모리**와 **프로젝트 메모리**.

## 파일 구조

```
my-project/
└── .relay/
    ├── memory/                  # 세션을 넘어 지속되는 메모리 (git 커밋 권장)
    │   ├── project.md           # 아키텍처, 도메인, 기술 스택 요약
    │   ├── lessons.md           # 반복되는 실수, 중요한 의사결정 히스토리
    │   └── agents/
    │       ├── pm.md            # PM 개인 메모리
    │       ├── fe.md            # FE: 컴포넌트 구조, 컨벤션
    │       ├── be.md            # BE: API 패턴, DB 스키마
    │       ├── da.md            # DA: 이벤트/메트릭 분류 체계
    │       ├── designer.md      # Designer: UI 패턴, 디자인 시스템
    │       ├── qa.md            # QA: 반복되는 버그 패턴, 취약한 영역
    │       └── deployer.md      # Deployer: 배포 히스토리, 주의사항
    └── sessions/                # 세션별 요약 (gitignore됨; 자동 생성)
        ├── 2026-03-13-001/
        │   └── summary.md
        └── ...
```

## 세션 메모리 (SQLite)

세션 내 데이터는 SQLite에 저장됩니다:
- 메시지 (`messages` 테이블)
- 태스크 (`tasks` 테이블)
- 아티팩트 (`artifacts` 테이블)
- 리뷰 (`reviews` 테이블)
- WebSocket 이벤트 (`events` 테이블)

세션이 종료되면 `save_session_summary`가 사람이 읽을 수 있는 요약을 `.relay/sessions/{session_id}/summary.md`에 저장합니다. 태스크와 메시지의 전체 기록은 이미 SQLite에 보존됩니다. sessions 디렉토리는 gitignore되어 있으며 로컬 전용 임시 데이터입니다.

## 프로젝트 메모리 (마크다운 파일)

세션을 넘어 지속되는 메모리는 `.relay/memory/`의 마크다운 파일로 저장됩니다.

### project.md

`/relay:init` 중에 코디네이터가 작성하는 프로젝트 전체 요약:
- 도메인 및 핵심 기능
- 아키텍처 개요
- 팀 컨벤션

### lessons.md

공유 팀 회고 및 의사결정 히스토리:
- 반복되는 실수 패턴
- 중요한 기술적 결정사항
- 특별히 주의가 필요한 영역

### agents/&#123;agentId&#125;.md

각 에이전트의 개인 메모리:
- 역할별로 중요한 코드 패턴
- 이전 세션에서 발견한 컨벤션
- 특정 영역에 대한 주의사항

## 메모리 MCP 도구

### 읽기

```typescript
// 에이전트 개인 메모리 읽기
read_memory({ agent_id: "fe" })
// → .relay/memory/agents/fe.md 반환

// 프로젝트 메모리 읽기 (agent_id 생략)
read_memory({})
// → project.md + lessons.md를 합쳐서 반환
```

### 쓰기 (섹션 덮어쓰기)

```typescript
write_memory({
  agent_id: "fe",
  key: "component-patterns",
  content: "Button components always use shadcn/ui as the base..."
})
// .relay/memory/agents/fe.md의 ## component-patterns 섹션을 교체
```

### 추가 (누적)

```typescript
// 에이전트 개인 메모리에 추가
append_memory({
  agent_id: "qa",
  content: "2026-03-13: found missing empty-email case in login flow"
})

// 팀 메모리에 추가 (lessons.md) — agent_id 생략
append_memory({
  content: "Always confirm BE auth header spec before FE implementation starts"
})
```

## 세션 생명주기

### 세션 시작 시

각 에이전트의 시스템 프롬프트에 자동으로 주입됩니다:
1. `project.md` + `lessons.md` (프로젝트 메모리)
2. `agents/{agentId}.md` (개인 메모리)

### 세션 종료 시

1. 각 에이전트가 `write_memory` 또는 `append_memory`로 새로 발견한 내용을 저장
2. 팀 회고를 `lessons.md`에 추가
3. `save_session_summary`로 세션을 아카이브

## 팀 공유

`.relay/memory/` 파일을 git에 커밋하면 팀 전체가 공유할 수 있어요:

```bash
git add .relay/memory/
git commit -m "chore: update relay memory"
```

이렇게 하면 새 팀원이나 새 Claude Code 세션이 이전 세션의 컨텍스트를
즉시 활용할 수 있습니다.

## RELAY_DIR 환경 변수

기본 메모리 경로는 `{cwd}/.relay`이지만, `RELAY_DIR` 환경 변수로 변경할 수 있습니다:

```bash
RELAY_DIR=/path/to/shared/.relay bun run start
```
