특정 에이전트의 전문성만 필요할 때 사용합니다. 팀 구성이나 조율 없이 바로 실행됩니다.

## 사용법

```
/relay:agent {agentId} "{지시사항}"
```

예시:

```
/relay:agent fe "CartItem 컴포넌트를 리팩터링해줘"
/relay:agent be "사용자 인증 API 보안 검토해줘"
/relay:agent qa "로그인 기능의 엣지 케이스 테스트를 작성해줘"
/relay:agent designer "체크아웃 플로우 UX 개선안을 작성해줘"
```

## /relay:relay 대신 사용하는 경우

- 특정 파일의 코드 리뷰나 빠른 피드백이 필요할 때
- 아키텍처 분석이나 기술 조언이 필요할 때
- 테스트, 문서, 스펙 같은 산출물 하나만 필요할 때
- 전체 팀을 띄울 만큼 범위가 크지 않을 때

## 사용 가능한 에이전트

에이전트는 풀 파일(`.relay/agents.pool.yml`)에서 가져옵니다. 존재하지 않는 ID를 지정하면 사용 가능한 목록을 보여줍니다.

## 에이전트에게 주입되는 컨텍스트

에이전트는 전체 프로젝트 컨텍스트를 갖고 작업합니다. 이전 세션의 개인 메모리와 공유 `project.md` 요약이 시스템 프롬프트에 자동 주입됩니다.
