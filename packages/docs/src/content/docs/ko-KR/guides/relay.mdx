메인 커맨드입니다. 목표만 알려주면 relay가 팀을 꾸리고, 에이전트들이 동시에 작업해 완료합니다.

## 사용법

```
/relay:relay "만들거나 수정할 내용 설명"
```

예시:

```
/relay:relay "장바구니 기능을 추가해줘"
/relay:relay "사용자 프로필 페이지를 개선해줘"
/relay:relay "결제 API 보안을 강화해줘"
```

## 제로 컨피그 첫 실행

`.relay/agents.pool.yml` 없이 `/relay:relay`를 실행하면 relay가 자동으로 다음을 수행합니다:

1. **프로젝트 분석** — README, package.json, 디렉토리 구조, 설정 파일을 읽습니다
2. **맞춤형 에이전트 풀 생성** — 세 가지 레인에서 4~8명을 선택합니다:
   - **코디네이션**: PM (항상 포함)
   - **구현**: FE, BE, mobile, devops 등 감지된 기술 스택 기반
   - **품질**: QA (테스트 러너 감지 시), security (인증/결제 감지 시)
3. **`.relay/agents.pool.yml` 작성** — 자유롭게 수정하거나, 삭제 후 다시 생성할 수 있습니다

생성된 프롬프트에는 프로젝트의 실제 파일 경로, 명령어, 컨벤션이 반영됩니다. 생성이 끝나면 곧바로 팀 구성 단계로 넘어갑니다.

<Aside type="tip">
  풀을 다시 생성하려면 `.relay/agents.pool.yml`을 삭제하고 `/relay:relay`를 다시 실행하세요.
</Aside>

## 실행하면 어떻게 되나요?

### 1. 팀 구성

에이전트를 실행하기 전, relay가 태스크 유형을 파악하고 풀에서 3~6명으로 팀을 제안합니다. 직접 조정할 수도 있습니다 — 전문가를 추가하거나, 필요 없는 역할을 빼거나, 같은 역할을 여러 명 넣을 수도 있습니다(예: 서로 리뷰하는 FE 두 명).

확정하면 세션이 시작됩니다.

### 1b. 계획 단계 (선택)

태스크가 복잡하면 전체 팀을 띄우기 전에 PM만 먼저 실행하는 사전 계획 단계를 요청할 수 있습니다. PM이 요구사항을 태스크로 쪼개두면, 나머지 에이전트들이 빈 보드가 아닌 명확한 태스크 보드를 갖고 시작합니다. 이 단계는 선택 사항이며, 전체 팀이 세션 시작과 함께 계획을 세울 수도 있습니다.

### 2. 모든 에이전트 동시 실행

팀의 모든 에이전트가 동시에 시작합니다. 코디네이터가 원래 요청을 받아 태스크로 분해하고, 나머지 에이전트들은 즉시 메시지와 태스크를 확인합니다. 순서를 기다리지 않습니다.

각 에이전트가 사용하는 정보:

- `agents.pool.yml`에 정의된 역할 페르소나
- 이전 세션의 개인 메모리 (`read_memory`)
- 공유 프로젝트 컨텍스트 (`project.md`)

### 3. 이벤트 기반 협업

에이전트끼리 직접 호출하지 않습니다. 모든 소통은 MCP 도구를 통해 이루어집니다:

- `send_message` / `get_messages` — 메시지 전달
- `create_task` / `claim_task` — 원자적 태스크 할당 (같은 태스크를 두 에이전트가 동시에 가져가지 않음)
- `post_artifact` / `get_artifact` — 산출물 공유
- `request_review` / `submit_review` — 피어 리뷰

리뷰가 필요하면 에이전트가 리뷰어를 지명합니다. 리뷰어가 실행되고, 리뷰를 마치면 원래 에이전트가 재개합니다. 오케스트레이터 없이 P2P로 진행됩니다.

### 4. 세션 종료

모든 에이전트가 작업을 마치면 세션이 아카이브되고 결과가 보고됩니다.

## 안정성 기능

### 오케스트레이터 상태 유지

오케스트레이터는 매 반복마다 `save_orchestrator_state`로 이벤트 루프 상태를 저장합니다. 재실행 시(예: 컨텍스트 압축 후) `get_orchestrator_state`로 `dormant_agents`, `done_agents`, `last_seen_msg_id` 등 전체 루프 상태를 복원하여, 메시지를 처음부터 다시 처리하지 않고 정확히 이전 지점에서 재개합니다.

<Aside type="tip">
  세션 시작 시 출력되는 대시보드 URL에서 진행 상황을 실시간으로 확인할 수 있습니다. 자세한 내용은
  [대시보드](/relay/ko-kr/guides/dashboard)를 참고하세요.
</Aside>

## 여러 인스턴스 동시 실행

여러 프로젝트를 동시에 작업할 때, relay 인스턴스를 여러 개 병렬로 실행할 수 있습니다:

```json
{
  "mcpServers": {
    "relay": { "env": { "DASHBOARD_PORT": "3456", "RELAY_INSTANCE": "project-a" } },
    "relay-b": { "env": { "DASHBOARD_PORT": "3457", "RELAY_INSTANCE": "project-b" } }
  }
}
```

각 인스턴스는 별도의 포트, 대시보드, 독립된 in-memory 저장소를 갖습니다.
