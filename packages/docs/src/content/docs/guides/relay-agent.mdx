`/relay:agent`는 특정 에이전트 한 명만 단독으로 호출한다.
전체 워크플로가 아닌 단일 에이전트의 전문성이 필요할 때 사용한다.

## 사용법

```
/relay:agent {에이전트ID} "{지시}"
```

예시:
```
/relay:agent fe "CartItem 컴포넌트 리팩토링해줘"
/relay:agent be "유저 인증 API 보안 검토해줘"
/relay:agent qa "로그인 기능 엣지케이스 테스트 시나리오 작성해줘"
/relay:agent designer "결제 플로우 UX 개선안 작성해줘"
```

## 사용 가능한 에이전트 ID

사용 가능한 에이전트 목록은 `agents.yml` 설정에 따라 다르다.
`list_agents` 호출 결과에서 ID 목록을 확인할 수 있다.

## 실행 과정

1. `list_agents`로 사용 가능한 에이전트 목록 확인
2. 대시보드 URL 안내: `http://localhost:3456`
3. 지정한 에이전트의 페르소나 + 기억 로드
4. 에이전트의 `tools` 배열로 MCP 툴 제한하여 단독 spawn
5. 작업 완료 후 `append_memory`로 학습 내용 저장

## 메모리 로드

단일 에이전트도 전체 컨텍스트를 가지고 작업한다:
1. `read_memory(agent_id: "{agentId}")` — 해당 에이전트의 개인 기억
2. `read_memory()` — `project.md` + `lessons.md` 프로젝트 기억

두 결과를 합성하여 system prompt에 prepend한다.

## 알 수 없는 에이전트 처리

지정한 에이전트 ID가 `list_agents` 결과에 없으면:
- 사용 가능한 에이전트 목록을 사용자에게 표시
- 다시 에이전트를 선택하도록 요청

## 활용 예시

**단일 에이전트 활용이 적합한 경우:**
- 특정 파일의 코드 리뷰가 필요할 때
- 아키텍처 분석이나 기술 조언이 필요할 때
- 전체 워크플로 없이 특정 산출물(테스트, 문서)만 필요할 때
- 빠른 작업으로 전체 팀을 동원하기 과한 경우
