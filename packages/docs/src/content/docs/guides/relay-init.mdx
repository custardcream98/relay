`/relay:init`은 프로젝트에 처음 relay를 사용하거나 팀이 프로젝트 컨텍스트를 새로 파악해야 할 때 실행한다.

`.relay/memory/`가 없으면 `/relay:relay` 실행 시 자동으로 이 스킬 실행을 제안한다.

## 실행 조건

- relay MCP 서버가 연결되어 있어야 한다
- 최초 1회 필수, 이후 프로젝트 구조가 크게 바뀌었을 때 재실행 권장

## Phase 0: 팀 제안 (agents.yml 없을 때)

`list_agents`가 빈 배열을 반환하면 아직 `agents.yml`이 없는 상태다.
프로젝트를 분석하여 도메인에 맞는 팀을 제안하고, `agents.yml`을 자동 생성한다.

생성 후에는 MCP 서버를 재시작한 다음 `/relay:init`을 다시 실행해야 한다.

## Phase 1: 병렬 프로젝트 스캔

`list_agents`에서 반환된 **모든 에이전트가 동시에** spawn되어 각자의 관점에서 코드베이스를 탐색한다.

각 에이전트는 자신의 `systemPrompt`에 정의된 역할 관점에서 탐색하고,
완료 후 `write_memory` 툴로 기억을 저장한 다음 브로드캐스트 메시지 `"init-done"`을 보낸다.

## Phase 2: 교차 검증

모든 에이전트의 `"init-done"` 브로드캐스트가 수집되거나 최대 5분 경과 시 다음 단계로 진행한다.

코디네이터 역할 에이전트(설명이나 이름에 "manager", "coordinator", "pm" 등 포함)가
각 에이전트의 기억을 읽고 `write_memory(key: "summary", ...)`로 `project.md`에 통합 요약을 저장한다.

## Phase 3: 완료 보고

사용자에게 init 결과를 요약 보고한다:
- 로드된 에이전트 목록
- 저장된 메모리 키 목록
- 이후 `/relay:relay "태스크"` 로 바로 시작 가능

## 출력 결과

init 완료 후 `.relay/memory/` 디렉토리가 생성된다:

```
.relay/
└── memory/
    ├── project.md               # 코디네이터가 작성한 통합 요약
    └── agents/
        ├── {agentId}.md         # 각 에이전트의 개인 기억 (agents.yml에 정의된 에이전트 수만큼)
        └── ...
```

이 파일들은 git에 커밋하여 팀이 공유하는 것을 권장한다.
