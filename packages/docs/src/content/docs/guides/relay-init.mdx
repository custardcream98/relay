---
title: /relay:init — project scan
description: /relay:init spawns all agents in parallel to scan your codebase and write role-specific notes to .relay/memory/ — run once per project for a shared context baseline.
---

Run `/relay:init` the first time you use relay in a project, or when the team needs to re-learn the project context.

If `.relay/memory/` does not exist, running `/relay:relay` will automatically suggest running this skill first.

## When to run

- relay MCP server must be connected
- Required once per project; re-run is recommended when the project structure changes significantly

## Phase 0: team suggestion (when no pool is configured)

If `list_agents` returns an empty array, no agent pool has been configured yet.
relay analyses the project, proposes a suitable team for the domain, and guides the user to create `.relay/agents.pool.yml`.

After the file is created, restart the MCP server and then run `/relay:init` again.

## Phase 1: parallel project scan

**All agents returned by `list_agents` spawn simultaneously** and explore the codebase from their own perspective.

Each agent scans from the role defined in its `systemPrompt`,
then saves its memory with `write_memory` and broadcasts `"init-done"`.

## Phase 2: cross-review

Once all agents' `"init-done"` broadcasts are collected, or after a maximum of 5 minutes, the process moves to the next phase.

The coordinator agent (one whose description or name contains "manager", "coordinator", or "pm")
reads each agent's memory and saves a consolidated summary to `project.md` via `write_memory(key: "summary", ...)`.

## Phase 3: completion report

Reports the init results to the user:
- List of loaded agents
- List of saved memory keys
- Ready to start with `/relay:relay "task"` immediately

## Output

After init completes, the `.relay/memory/` directory is created:

```
.relay/
└── memory/
    ├── project.md               # consolidated summary written by the coordinator
    └── agents/
        ├── {agentId}.md         # each agent's personal memory (one per agent in the pool)
        └── ...
```

It is recommended to commit these files to git so the team shares them.
