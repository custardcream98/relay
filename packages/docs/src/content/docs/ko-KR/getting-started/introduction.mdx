기능을 설명하면, AI 에이전트 팀이 함께 만들어냅니다.

그게 relay입니다. 순서를 기다리지 않습니다. 단계가 없습니다. 병렬로 작업하는 진짜 팀입니다.

## 단일 에이전트 AI의 한계

대부분의 AI 코딩 도구는 하나의 에이전트가 모든 것을 합니다: 계획, 조사, 코딩, 리뷰, 테스트, 배포 — 순차적으로, 하나의 컨텍스트 윈도우 안에서. 단순한 작업에는 충분합니다. 복잡한 작업에서는 한계를 드러냅니다.

단일 에이전트는 전문화할 수 없고, 병렬로 처리할 수 없으며, 자신의 실수를 잡아낼 수 없습니다.

## relay는 다릅니다

relay는 Claude Code 서브에이전트 팀을 생성합니다 — 각자의 역할, 각자의 페르소나, 각자의 판단을 가진 팀을. 에이전트들은 동시에 실행되고, 공유 MCP 서버를 통해 소통하며, 작업이 완료될 때까지 P2P로 협업합니다.

단계가 없습니다. 순서를 기다리지 않습니다. 병렬로 작업하는 팀이 있을 뿐입니다.

## 실시간으로 지켜보세요

`http://localhost:3456`의 실시간 대시보드가 에이전트 작업에 따라 WebSocket으로 업데이트됩니다:

- **세션 진행 위젯** — 헤더에서 태스크 완료 비율, 활성 에이전트 수, 경과 시간을 한눈에 표시
- **Agent Arena** — 모든 에이전트를 나란히 표시 — 실시간 상태, 현재 생각 스니펫, 카드별 태스크 완료 미니 바
- **Activity Feed** — 메시지, 추론 스트림, 태스크 변경, 아티팩트를 하나의 뷰에 통합한 타임라인. 키보드 단축키(`j`/`k`/`Enter`/`Escape`)로 탐색 가능
- **Task Board** — 색상별 프로그레스 바와 의존성 표시("Blocked by N" / "Blocks N tasks")가 있는 접기 가능한 Kanban
- **모바일 레이아웃** — 좁은 화면에서 Agents / Activity / Tasks 탭의 하단 내비게이션

세션 데이터는 서버 프로세스가 살아있는 동안 in-memory로 유지됩니다.

## 어떤 팀이든, 어떤 도메인이든

relay에는 기본 제공 에이전트가 없습니다. `.relay/agents.pool.yml`에 에이전트 풀을 정의하면 relay가 각 태스크에 최적화된 팀을 선발합니다:

```yaml
agents:
  pm:
    name: Project Manager
    emoji: "📋"
    tags: [planning, coordination]
    tools: [create_task, get_all_tasks, send_message, get_messages]
    systemPrompt: |
      당신은 프로젝트 매니저입니다. 요구사항을 태스크로 분해하세요.

  researcher:
    name: Researcher
    emoji: "🔬"
    tags: [research, analysis]
    tools: [claim_task, post_artifact, send_message]
    systemPrompt: |
      당신은 리서처입니다. 주제를 조사하고 결과를 아티팩트로 게시하세요.

  researcher2:
    extends: researcher  # 페르소나를 상속하고 필드를 오버라이드
    name: Senior Researcher
    emoji: "🔭"
```

웹 개발팀, 리서치팀, 마케팅팀, 법무팀 — 역할을 시스템 프롬프트로 설명할 수 있다면, relay가 실행할 수 있습니다.

## 구조

relay는 **Claude Code 플러그인**입니다. [Claude Code](https://claude.ai/download) v1.0.33 이상이 필요하며, Claude Code 안에서 `/plugin marketplace add`와 `/plugin install`로 설치합니다. 자세한 내용은 [설치 가이드](/relay/ko-kr/getting-started/installation/)를 참고하세요.

relay는 세 가지 레이어로 구성됩니다:

| 레이어 | 역할 |
|---|---|
| **MCP server** | 메시지 버스, 태스크 보드, 아티팩트 저장소, 메모리 레이어. Claude API 호출 없음 — 순수 인프라. |
| **Skills** | Claude Code에게 에이전트를 어떻게 오케스트레이션할지 알려주는 `.md` 파일. 텍스트 편집으로 동작을 변경합니다. |
| **Hooks** | `PostToolUse` (대시보드 푸시), `PreToolUse` (편집 가드), `Stop` (오케스트레이터 정리), `SessionEnd` (비동기 세션 정리). |

<Aside type="tip">
relay는 Claude Code의 내장 Agent 도구만 사용합니다. Claude API를 직접 호출하지 않으므로 기존 Claude Code 구독 외에 추가 비용이 발생하지 않습니다.
</Aside>

## 무엇을 얻을 수 있나요

- **병렬 실행** — 모든 에이전트가 동시에 시작, 순차적 단계 없음
- **원자적 태스크 클레임** — 에이전트들이 태스크를 경쟁적으로 클레임; `claim_task`는 레이스 컨디션 안전
- **영속 메모리** — 에이전트가 세션 종료 후 `.relay/memory/`에 학습 내용 기록; 다음 세션이 이어받음
- **실시간 대시보드** — 태스크, 메시지, 에이전트 추론이 `http://localhost:3456`에서 실시간 업데이트
- **세션 요약 아카이브** — 각 세션이 `.relay/sessions/`에 저장; 에이전트가 `get_session_summary`로 이전 컨텍스트를 불러올 수 있음
