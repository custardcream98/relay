title: /relay:init — 프로젝트 스캔
description: 팀 전체가 코드베이스를 병렬로 스캔하고 프로젝트 메모리를 초기화하는 Skill
---

프로젝트에서 relay를 처음 사용할 때, 또는 팀이 프로젝트 컨텍스트를 다시 학습해야 할 때 `/relay:init`을 실행하세요.

`.relay/memory/`가 없는 상태에서 `/relay:relay`를 실행하면, 먼저 이 Skill을 실행하도록 자동으로 안내합니다.

## 실행 조건

- relay MCP 서버가 연결되어 있어야 합니다
- 프로젝트당 최초 1회 실행이 필요하며, 프로젝트 구조가 크게 변경된 경우 재실행을 권장합니다

## Phase 0: 팀 제안 (agents.yml이 없을 때)

`list_agents`가 빈 배열을 반환하면 `agents.yml`이 아직 생성되지 않은 것입니다.
relay가 프로젝트를 분석해 도메인에 적합한 팀을 제안하고, `agents.yml`을 자동 생성합니다.

파일이 생성된 후 MCP 서버를 재시작하고 `/relay:init`을 다시 실행하세요.

## Phase 1: 병렬 프로젝트 스캔

**`list_agents`가 반환한 모든 에이전트가 동시에 실행**되어 각자의 관점에서 코드베이스를 탐색합니다.

각 에이전트는 `systemPrompt`에 정의된 역할에 따라 스캔을 수행한 후,
`write_memory`로 메모리를 저장하고 `"init-done"`을 브로드캐스트합니다.

## Phase 2: 교차 검토

모든 에이전트의 `"init-done"` 브로드캐스트가 수집되거나, 최대 5분이 지나면 다음 단계로 넘어갑니다.

코디네이터 에이전트(description이나 name에 "manager", "coordinator", "pm"이 포함된 에이전트)가
각 에이전트의 메모리를 읽고, `write_memory(key: "summary", ...)`를 통해 통합 요약을 `project.md`에 저장합니다.

## Phase 3: 완료 리포트

init 결과를 사용자에게 리포트합니다:
- 로드된 에이전트 목록
- 저장된 메모리 키 목록
- `/relay:relay "태스크"`로 바로 시작할 수 있음을 안내

## 출력 결과

init이 완료되면 `.relay/memory/` 디렉터리가 생성됩니다:

```
.relay/
└── memory/
    ├── project.md               # 코디네이터가 작성한 통합 요약
    └── agents/
        ├── {agentId}.md         # 각 에이전트의 개인 메모리 (agents.yml의 에이전트마다 생성)
        └── ...
```

이 파일들을 git에 커밋해두면 팀 전체가 공유할 수 있어요.
