`agents.yml`은 에이전트 페르소나와 워크플로를 커스터마이즈하는 설정 파일이다.
`agents.default.yml`의 기본값을 오버라이드한다.

## 최상위 구조

```yaml
agents:
  {agentId}:
    # 에이전트 설정
    ...

workflow:
  jobs:
    # 워크플로 설정
    ...
```

## agents 섹션

### 에이전트 스키마

```yaml
agents:
  {agentId}:
    name: string           # 표시 이름 (예: "Frontend Engineer")
    emoji: string          # 이모지 아이콘 (예: "💻")
    description: string    # 짧은 역할 설명
    disabled: boolean      # true이면 비활성화 (기본값: false)
    extends: string        # 상속받을 에이전트 ID
    tools:                 # 사용 가능한 MCP 툴 목록
      - tool_name
    systemPrompt: string   # 에이전트 system prompt (멀티라인)
```

### 기본 에이전트 ID

| ID | 이름 |
|---|---|
| `pm` | Product Manager |
| `designer` | UX Designer |
| `da` | Data Analyst |
| `fe` | Frontend Engineer |
| `be` | Backend Engineer |
| `qa` | QA Engineer |
| `deployer` | Deployer |

## workflow 섹션

### job 스키마

```yaml
workflow:
  jobs:
    {jobId}:
      agents:              # 이 job에서 병렬로 실행될 에이전트 ID 목록
        - agentId
      description: string  # job 설명 (에이전트 system prompt에 주입됨)
      reviewers:           # 리뷰어 매핑 (선택)
        {agentId}: [reviewerAgentId]
      end:                 # 다음 job 결정 조건
        {nextJobId}: "조건 설명"
        _done: "워크플로 종료 조건"
```

### 특수 job ID

- `_done`: 워크플로 종료를 나타내는 예약어

### 시작 job 결정

시작 job은 자동으로 결정된다: 어떤 job의 `end` 목적지에도 지정되지 않은 job이 시작점이다.

## 전체 예시

```yaml
agents:
  fe:
    name: "Frontend Engineer (Next.js)"
    systemPrompt: |
      당신은 Next.js App Router 전문 Frontend Engineer입니다.
      TypeScript, Tailwind CSS, React Query를 주로 사용합니다.

      ## 역할
      - 디자인 스펙을 바탕으로 Next.js 컴포넌트를 구현합니다
      - PR 아티팩트를 생성하고 FE2에게 코드리뷰를 요청합니다

      ## 규칙
      - agent_id는 "fe" 또는 "fe2"로 설정 (리뷰어 역할 시 "fe2")

  security:
    name: "Security Engineer"
    emoji: "🔐"
    description: "보안 취약점 분석"
    tools:
      - get_messages
      - get_artifact
      - post_artifact
      - send_message
    systemPrompt: |
      당신은 보안 전문가입니다.
      OWASP Top 10 기준으로 PR을 검토합니다.

  da:
    disabled: true  # DA 에이전트 비활성화

workflow:
  jobs:
    planning:
      agents: [pm]
      description: "요구사항 분석 및 태스크 분해"
      end:
        development: "태스크 배정 완료 시"

    development:
      agents: [fe, be]
      description: "구현"
      end:
        security-review: "구현 완료 시"

    security-review:
      agents: [security]
      description: "보안 검토"
      end:
        qa: "보안 이슈 없으면"
        development: "수정 필요 시"

    qa:
      agents: [qa]
      description: "기능 테스트"
      end:
        deploy: "테스트 통과 시"
        development: "버그 발견 시"

    deploy:
      agents: [deployer]
      description: "배포"
      end:
        _done: "배포 완료 시"
```
