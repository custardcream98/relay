설치 후 처음으로 relay를 프로젝트에 적용하는 가이드입니다.

<Steps>

1. **프로젝트 스캔 (프로젝트당 최초 1회)**

   새 프로젝트에 relay를 처음 사용하거나, 팀이 코드베이스를 다시 학습해야 할 때 실행하세요:

   ```
   /relay:init
   ```

   설정된 모든 에이전트가 병렬로 실행되어 각자의 관점에서 코드베이스를 스캔하고,
   프로젝트 메모리를 `.relay/memory/`에 저장합니다.

2. **태스크 실행**

   init이 완료되면 즉시 태스크를 요청할 수 있어요:

   ```
   /relay:relay "장바구니 기능을 추가해줘"
   ```

   에이전트들이 설정된 워크플로우에 따라 이벤트 드리븐으로 협업합니다.

3. **대시보드 확인 (선택사항)**

   MCP 서버가 실행되면 웹 대시보드에서 에이전트 활동을 실시간으로 모니터링할 수 있어요:

   ```
   http://localhost:3456
   ```

4. **단일 에이전트 호출 (필요할 때)**

   특정 에이전트의 전문성만 필요한 경우:

   ```
   /relay:agent fe "CartItem 컴포넌트를 리팩터링해줘"
   /relay:agent be "사용자 인증 API 보안 검토해줘"
   ```

</Steps>

## 실행 흐름 예시

```
사용자: /relay:relay "로그인 기능을 구현해줘"

[PM]       → 로그인을 5개의 태스크로 분해
             → FE: UI, BE: API, Designer: 화면 스펙, DA: 이벤트 설계

[Designer] → 로그인 화면 스펙 작성 (post_artifact)
[DA]       → login_success, login_fail 이벤트 설계 (post_artifact)

[FE]       → 로그인 UI 구현, fe-pr 아티팩트 생성
[BE]       → 인증 API 구현, be-pr 아티팩트 생성

[FE2]      → FE PR 리뷰 (request_review → submit_review)
[BE2]      → BE PR 리뷰

[QA]       → 테스트 시나리오 작성 및 검증
           → 발견된 버그는 티켓 생성 → 개발팀이 수정

[Deployer] → 배포 완료, 팀에 알림

사용자: ✅ 로그인 기능 배포 완료!
```
