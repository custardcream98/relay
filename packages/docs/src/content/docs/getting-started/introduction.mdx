title: What is relay?
description: A domain-agnostic multi-agent collaboration framework built on Claude Code
---

relay is a **domain-agnostic** multi-agent framework where Claude Code sub-agents collaborate under their own personas.
Define any team you want in `agents.yml` — a web dev team, research team, marketing team, legal team, or anything else.

Uses only Claude Code's Agent tool — no extra API billing.

## How it works

```
User: "Add a shopping cart feature"
  ↓
[PM]       → break down tasks, create issues
[Designer] → UX flow, component spec
[DA]       → event design, KPI definition
[BE]       → API design and implementation
[FE]       → UI implementation
[FE2]      → FE code review
[BE2]      → BE code review
[QA]       → test scenarios and bug reports
[Deployer] → deploy
```

## Three layers

| Layer | Role |
|---|---|
| MCP server | Inter-agent communication infrastructure (message bus, task board, memory) |
| Skills | `/relay:relay`, `/relay:init`, `/relay:agent` orchestration strategies |
| Hooks | PostToolUse — real-time dashboard updates on MCP tool calls |

## Agents are defined by you

Agent configuration is fully customisable via `agents.yml`.
`agents.example.yml` includes a web dev team example (pm, designer, da, fe, be, qa, deployer).
Copy it to get started, or define your own team from scratch.

## Why relay?

Most AI agent tools run everything through a single agent. relay is different.

Each agent handles only its own role, communicating directly with others through the MCP server.
Peer-to-peer — agents collaborate event-driven, with no central orchestrator.
Uses only Claude Code's Agent tool — no extra API billing.
