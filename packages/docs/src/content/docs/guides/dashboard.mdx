The relay MCP server also serves a web dashboard.
You can watch agents think and communicate with each other in real time.

## Accessing

The dashboard starts automatically when the MCP server starts:

```
http://localhost:3456
```

## Three panels

![relay dashboard — live session in progress](/relay/screenshots/dashboard-en.png)

### Task Board (Kanban)

- Displays the full issue status as a Kanban board
- Updates instantly when task state changes: `todo` → `in_progress` → `in_review` → `done`
- Click a task to view details (assignee, priority, description)

### Message Feed

- Displays inter-agent conversations as Slack-style threads
- Distinguishes `to: null` broadcast messages from direct messages
- New messages appear in real time

### Agent Thoughts

- Streams the selected agent's reasoning (text before tool calls) in real time
- Agent status indicator: `idle` (grey), `working` (green), `waiting` (yellow)

## WebSocket events

The dashboard receives real-time updates over WebSocket.
All events follow the `{ type, timestamp, ...payload }` structure — there is no separate `payload` wrapper field.

| Event type | Description |
|---|---|
| `agent:thinking` | Agent reasoning text stream |
| `agent:status` | Agent status change (idle/working/waiting) |
| `message:new` | New message received |
| `task:updated` | Task state change |
| `artifact:posted` | New artifact shared |
| `review:requested` | Review request raised |
| `memory:updated` | Agent memory updated |

## History replay

All events are stored in SQLite with timestamps.

Even after a session ends, you can select a session from the dashboard and replay the entire process:
- The order in which agents worked
- What messages were exchanged
- Where blockers occurred

## REST API

The dashboard also exposes a Hono-based REST API:

```
GET  /api/session               # current session snapshot (tasks, messages, artifacts)
GET  /api/sessions/:id          # specific session summary (summary.md)
GET  /api/sessions/:id/events   # session event list (for history replay)
GET  /api/agents                # agent list
POST /api/hook/tool-use         # PostToolUse hook receiver (internal)
```
