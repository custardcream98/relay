---
title: 에이전트 커스터마이즈
description: agents.yml로 에이전트 페르소나를 커스터마이즈하는 방법
---

relay의 에이전트 페르소나는 YAML 파일을 통해 완전히 커스터마이즈할 수 있습니다.

## 파일 구조

```
relay/
├── agents.default.yml   # 기본 페르소나 (수정하지 마세요)
└── agents.yml           # 커스터마이즈 파일 (이 파일을 수정하세요)
```

`agents.default.yml`은 수정하지 마세요. 모든 커스터마이즈는 `agents.yml`에서 합니다.
`agents.yml`이 `agents.default.yml`을 덮어씁니다.

## agents.yml 기본 구조

```yaml
agents:
  fe:
    # 변경할 필드만 지정하면 됩니다
    systemPrompt: |
      You are a React and TypeScript expert.
      ...
```

## 오버라이드

기존 에이전트의 특정 필드만 변경합니다:

```yaml
agents:
  fe:
    name: "Frontend Engineer (React specialist)"
    systemPrompt: |
      You are a Frontend Engineer at a startup.
      Your primary stack is React, TypeScript, and Tailwind CSS.
      ...
```

## extends

다른 에이전트의 설정을 상속해 확장합니다:

```yaml
agents:
  fe-mobile:
    extends: fe          # fe 페르소나를 상속
    name: "Mobile Frontend Engineer"
    systemPrompt: |
      You are a Frontend Engineer specialising in React Native.
      ...
```

## disabled

특정 에이전트를 비활성화합니다:

```yaml
agents:
  da:
    disabled: true       # DA 에이전트 비활성화
```

비활성화된 에이전트는 `list_agents` 결과에서 제외되고 모든 워크플로우에서 사용되지 않습니다.

## 커스텀 에이전트 추가

기본 에이전트 외에 완전히 새로운 페르소나를 추가할 수 있습니다:

```yaml
agents:
  security:
    name: "Security Engineer"
    emoji: "🔐"
    description: "Security vulnerability analysis and remediation"
    tools:
      - get_messages
      - get_my_tasks
      - update_task
      - get_artifact
      - post_artifact
      - send_message
    systemPrompt: |
      You are a security expert.
      Review code against the OWASP Top 10 and
      identify vulnerabilities with recommended fixes.
```

## tools 필드

에이전트가 호출할 수 있는 MCP 도구 목록을 지정합니다:

```yaml
tools:
  - send_message
  - get_messages
  - create_task
  - update_task
  - get_my_tasks
  - post_artifact
  - get_artifact
  - request_review
  - submit_review
  - read_memory
  - write_memory
  - append_memory
```

`tools` 배열에 없는 MCP 도구는 에이전트가 호출할 수 없습니다.
각 역할에 필요한 최소한의 도구만 부여하는 것을 권장합니다.
