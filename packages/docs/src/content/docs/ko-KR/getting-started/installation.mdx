relay는 **Claude Code 플러그인**입니다. 설치하면 Claude Code가 자동으로 설정합니다:

- 슬래시 커맨드 3개: `/relay:relay`, `/relay:init`, `/relay:agent`
- 에이전트 활동을 실시간 대시보드로 스트리밍하는 `PostToolUse` 훅
- 에이전트 간 통신을 처리하는 MCP 서버

## 사전 요구사항

- [Node.js](https://nodejs.org) v18 이상
- [Claude Code](https://claude.ai/download) v1.0.33 이상 (`claude --version`으로 확인)

## 1단계 — relay 마켓플레이스 추가

Claude Code는 마켓플레이스를 통해 플러그인을 설치합니다. GitHub 저장소에서 relay 마켓플레이스를 추가합니다:

```
/plugin marketplace add custardcream98/relay
```

이 명령은 카탈로그를 등록합니다. 아직 플러그인이 설치된 것은 아닙니다.

## 2단계 — 플러그인 설치

relay 플러그인을 user 스코프로 설치합니다 (모든 프로젝트에서 사용 가능):

```
/plugin install relay@relay
```

현재 프로젝트에만 설치하려면 (저장소를 신뢰하는 팀원과 공유됨):

```
/plugin install relay@relay --scope project
```

설치 후 `/reload-plugins`를 실행하거나 Claude Code를 재시작하세요. 슬래시 커맨드 목록에 `/relay:relay`, `/relay:init`, `/relay:agent`가 표시되어야 합니다.

<Aside type="note">
Skills는 플러그인 이름으로 네임스페이스가 지정됩니다 — `/init`이 아니라 `/relay:init`. 다른 플러그인과의 충돌을 방지합니다.
</Aside>

## 3단계 — 에이전트 풀 구성

relay는 기본 에이전트를 제공하지 않습니다. 프로젝트 루트의 `.relay/agents.pool.yml`에 팀을 직접 정의합니다.

예시 풀을 복사해서 시작하세요:

```bash
curl -O https://raw.githubusercontent.com/custardcream98/relay/main/agents.pool.example.yml
mv agents.pool.example.yml .relay/agents.pool.yml
```

또는 `.relay/agents.pool.yml`을 직접 작성합니다. 최소 2인 팀 예시:

```yaml
agents:
  pm:
    name: Project Manager
    emoji: "📋"
    tags: [planning, coordination]
    tools: [create_task, get_all_tasks, send_message, get_messages, broadcast_thinking]
    systemPrompt: |
      You are the project manager. Break down the user's request into
      concrete tasks and assign them to the team via create_task.

  fe:
    name: Frontend Engineer
    emoji: "🎨"
    tags: [frontend, react, css]
    tools: [claim_task, update_task, post_artifact, send_message, broadcast_thinking]
    systemPrompt: |
      You are a frontend engineer. Claim frontend tasks and implement them.
```

전체 스키마, 멀티 인스턴스 에이전트(`extends`), 12인 팀 예시는 [에이전트 커스터마이즈](/relay/ko-kr/guides/agents/)를 참고하세요.

<Aside type="tip">
`.relay/agents.pool.yml`을 git에 커밋하세요. 팀 전원이 동일한 에이전트 구성을 공유합니다.
</Aside>

## 4단계 — 설치 확인

프로젝트 스캔을 실행해서 모든 구성이 제대로 됐는지 확인합니다:

```
/relay:init
```

설정된 모든 에이전트가 병렬로 실행되고, 코드베이스를 스캔하고, `.relay/memory/`에 학습한 내용을 기록합니다. 에이전트가 정상 시작되고 출력에 대시보드 URL이 표시되면 설치가 완료된 것입니다.
