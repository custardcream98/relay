relay의 에이전트 기억은 **세션 메모리**와 **프로젝트 메모리** 두 레이어로 분리된다.

## 파일 구조

```
my-project/
└── .relay/
    ├── memory/                  # 세션 간 영속 기억 (git 커밋 권장)
    │   ├── project.md           # 아키텍처, 도메인, 기술 스택 요약
    │   ├── lessons.md           # 반복 실수, 중요 의사결정 히스토리
    │   └── agents/
    │       ├── pm.md            # PM 전용 기억
    │       ├── fe.md            # FE: 컴포넌트 구조, 컨벤션
    │       ├── be.md            # BE: API 패턴, DB 스키마
    │       ├── da.md            # DA: 이벤트/지표 체계
    │       ├── designer.md      # Designer: UI 패턴, 디자인 시스템
    │       ├── qa.md            # QA: 반복 버그 패턴, 취약 영역
    │       └── deployer.md      # Deployer: 배포 히스토리, 주의사항
    └── sessions/                # 세션별 감사 로그 (자동 생성)
        ├── 2026-03-13-001/
        │   ├── messages.json
        │   ├── tasks.json
        │   └── summary.md
        └── ...
```

## 세션 메모리 (SQLite)

세션 내 데이터는 SQLite에 저장된다:
- 메시지 (`messages` 테이블)
- 태스크 (`tasks` 테이블)
- 아티팩트 (`artifacts` 테이블)
- 리뷰 (`reviews` 테이블)
- WebSocket 이벤트 (`events` 테이블)

세션이 종료되면 `save_session_summary`로 `.relay/sessions/`에 아카이브된다.

## 프로젝트 메모리 (Markdown 파일)

세션 간에 영속되는 기억은 `.relay/memory/`의 Markdown 파일로 저장된다.

### project.md

PM이 `/relay:init` 시 작성하는 프로젝트 전체 요약:
- 도메인 및 주요 기능
- 아키텍처 개요
- 팀 컨벤션

### lessons.md

팀 전체가 공유하는 회고 및 의사결정 히스토리:
- 반복되는 실수 패턴
- 중요한 기술적 의사결정
- 주의해야 할 영역

### agents/&#123;agentId&#125;.md

각 에이전트의 개인 기억:
- 역할별로 중요한 코드 패턴
- 이전 세션에서 발견한 컨벤션
- 특정 영역의 주의사항

## 메모리 MCP 툴

### 읽기

```typescript
// 에이전트 개인 기억 읽기
read_memory({ agent_id: "fe" })
// → .relay/memory/agents/fe.md 반환

// 프로젝트 기억 읽기 (agent_id 생략)
read_memory({})
// → project.md + lessons.md 합성 반환
```

### 쓰기 (섹션 덮어쓰기)

```typescript
write_memory({
  agent_id: "fe",
  key: "component-patterns",
  content: "Button 컴포넌트는 항상 shadcn/ui 기반으로..."
})
// .relay/memory/agents/fe.md의 ## component-patterns 섹션을 교체
```

### 추가 (누적 appending)

```typescript
// 에이전트 개인 기억에 추가
append_memory({
  agent_id: "qa",
  content: "2026-03-13: 로그인 flow에서 빈 이메일 케이스 누락 발견"
})

// 팀 기억(lessons.md)에 추가 (agent_id 생략)
append_memory({
  content: "auth 헤더는 반드시 BE 스펙 확인 후 FE 구현 시작할 것"
})
```

## 세션 생명주기

### 세션 시작 시

에이전트 spawn 시 system prompt에 자동 주입:
1. `project.md` + `lessons.md` (프로젝트 기억)
2. `agents/&#123;agentId&#125;.md` (개인 기억)

### 세션 종료 시

1. 각 에이전트가 `write_memory` 또는 `append_memory`로 새로운 발견 저장
2. 팀 공유 회고를 `lessons.md`에 추가
3. `save_session_summary`로 세션 아카이브

## 팀 협업

`.relay/memory/` 파일들은 git에 커밋하여 팀 전체가 공유하는 것을 권장한다:

```bash
git add .relay/memory/
git commit -m "chore: relay 메모리 업데이트"
```

이를 통해 새로운 팀원이나 새 Claude Code 세션에서도
이전 세션의 컨텍스트를 즉시 활용할 수 있다.

## RELAY_DIR 환경변수

기본 메모리 경로는 `{cwd}/.relay`이지만, `RELAY_DIR` 환경변수로 변경할 수 있다:

```bash
RELAY_DIR=/path/to/shared/.relay bun run start
```
