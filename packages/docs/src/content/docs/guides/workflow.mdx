---
title: Workflow configuration
description: Customise the task execution flow with the workflow section in agents.yml
---

The `workflow` section in `agents.yml` gives you full control over the order in which agents work.

## Basic structure

```yaml
workflow:
  jobs:
    {jobId}:
      agents: [list of agent IDs]
      description: "description of work done in this job"
      end:
        {nextJobId}: "advance to the next job when this condition is met"
        {anotherJobId}: "advance to another job when this other condition is met"
```

## Special values

- `_done`: reserved job ID that signals the end of the workflow
- If multiple `end` conditions are defined, the agent chooses based on the situation

## Default workflow example

```yaml
workflow:
  jobs:
    planning:
      agents: [pm]
      description: "Analyse user requirements and break down into tasks."
      end:
        design: "When all tasks have been created and assigned"

    design:
      agents: [designer, da]
      description: "Designer writes the UX spec; DA writes the analytics plan."
      end:
        development: "When both deliverables have been shared via post_artifact"

    development:
      agents: [fe, be]
      description: "FE implements the UI; BE implements the API. Both create PR artifacts."
      end:
        review: "When implementation is complete and PRs are ready"

    review:
      agents: [fe2, be2]
      reviewers:
        fe: [fe2]
        be: [be2]
      description: "Review the PRs."
      end:
        qa: "When all PRs are LGTM"
        development: "When changes are requested"

    qa:
      agents: [qa]
      description: "Test the feature and create tickets for any bugs found."
      end:
        deploy: "When all tests pass"
        development: "When bugs are found"

    deploy:
      agents: [deployer]
      description: "Deploy the QA-approved feature to production."
      end:
        _done: "When deployment is complete"
```

## reviewers field

Use the `reviewers` field in jobs that require review:

```yaml
review:
  agents: [fe2, be2]
  reviewers:
    fe: [fe2]    # fe2 reviews artifacts created by fe
    be: [be2]    # be2 reviews artifacts created by be
```

Reviewer agents are spawned with the persona of their corresponding author (fe, be),
but with `agent_id` set to `fe2` or `be2`.

## Simplified workflow

For quick prototypes and simple tasks, you can strip the workflow down:

```yaml
workflow:
  jobs:
    planning:
      agents: [pm]
      description: "Break down tasks"
      end:
        development: "When tasks are created"

    development:
      agents: [fe, be]
      description: "Implement"
      end:
        _done: "When implementation is complete"
```

## Parallel execution

Listing multiple agents in the `agents` array spawns them all simultaneously:

```yaml
design:
  agents: [designer, da]  # both agents work at the same time
```

All agents in the array must declare `end` before the workflow advances to the next job.
