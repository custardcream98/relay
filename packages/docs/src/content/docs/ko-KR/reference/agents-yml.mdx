`agents.yml`은 에이전트 페르소나와 워크플로우를 커스터마이즈하는 설정 파일입니다.
`agents.default.yml`의 기본값을 덮어씁니다.

## 최상위 구조

```yaml
agents:
  {agentId}:
    # 에이전트 설정
    ...

workflow:
  jobs:
    # 워크플로우 설정
    ...
```

## agents 섹션

### 에이전트 스키마

```yaml
agents:
  {agentId}:
    name: string           # 표시 이름 (예: "Frontend Engineer")
    emoji: string          # 이모지 아이콘 (예: "💻")
    description: string    # 짧은 역할 설명
    disabled: boolean      # true이면 비활성화 (기본값: false)
    extends: string        # 상속할 에이전트 ID
    tools:                 # 허용된 MCP 도구 목록
      - tool_name
    systemPrompt: string   # 에이전트 시스템 프롬프트 (여러 줄)
```

### 기본 에이전트 ID

| ID | 이름 |
|---|---|
| `pm` | Product Manager |
| `designer` | UX Designer |
| `da` | Data Analyst |
| `fe` | Frontend Engineer |
| `be` | Backend Engineer |
| `qa` | QA Engineer |
| `deployer` | Deployer |

## workflow 섹션

### job 스키마

```yaml
workflow:
  jobs:
    {jobId}:
      agents:              # 이 job에서 병렬로 실행할 에이전트 ID 목록
        - agentId
      description: string  # job 설명 (에이전트 시스템 프롬프트에 주입됨)
      reviewers:           # 리뷰어 매핑 (선택사항)
        {agentId}: [reviewerAgentId]
      end:                 # 다음 job을 선택하는 조건
        {nextJobId}: "조건 설명"
        _done: "워크플로우 종료 조건"
```

### 특수 job ID

- `_done`: 워크플로우의 끝을 나타내는 예약 키워드

### 시작 job 결정

시작 job은 자동으로 결정됩니다: 다른 job의 `end` 맵에 목적지로 나타나지 않는 job이 진입점입니다.

## 전체 예시

```yaml
agents:
  fe:
    name: "Frontend Engineer (Next.js)"
    systemPrompt: |
      You are a Frontend Engineer specialising in Next.js App Router.
      Your primary stack is TypeScript, Tailwind CSS, and React Query.

      ## Role
      - Implement Next.js components from design specs
      - Create PR artifacts and request code review from FE2

      ## Rules
      - Set agent_id to "fe" normally, or "fe2" when acting as a reviewer

  security:
    name: "Security Engineer"
    emoji: "🔐"
    description: "Security vulnerability analysis"
    tools:
      - get_messages
      - get_artifact
      - post_artifact
      - send_message
    systemPrompt: |
      You are a security expert.
      Review PRs against the OWASP Top 10.

  da:
    disabled: true  # DA 에이전트 비활성화

workflow:
  jobs:
    planning:
      agents: [pm]
      description: "Analyse requirements and break down into tasks"
      end:
        development: "When tasks are assigned"

    development:
      agents: [fe, be]
      description: "Implement"
      end:
        security-review: "When implementation is complete"

    security-review:
      agents: [security]
      description: "Security review"
      end:
        qa: "When no security issues are found"
        development: "When fixes are needed"

    qa:
      agents: [qa]
      description: "Feature testing"
      end:
        deploy: "When tests pass"
        development: "When bugs are found"

    deploy:
      agents: [deployer]
      description: "Deploy"
      end:
        _done: "When deployment is complete"
```
