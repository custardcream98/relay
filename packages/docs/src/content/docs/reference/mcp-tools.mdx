title: MCP tools reference
description: "Complete MCP tool reference for relay — parameters, return values, and usage for all 18 tools: messaging, tasks, artifacts, reviews, memory, sessions, and visibility."
---

All tools respond in the form `{ success: boolean, data?, error? }`.
`agent_id` is always included in parameters to track who called.
`session_id` is injected automatically by the server from the `RELAY_SESSION_ID` environment variable — clients do not need to pass it.

## Messaging

### send_message

Send a message between agents.

**Parameters:**
```typescript
{
  agent_id: string;    // sending agent ID
  to: string | null;   // receiving agent ID. null = broadcast
  content: string;     // message content
  thread_id?: string;  // thread ID (optional)
}
```

**Returns:**
```typescript
{
  success: true;
  message_id: string;
}
```

---

### get_messages

Fetch received messages (direct messages + broadcasts).

**Parameters:**
```typescript
{
  agent_id: string; // agent ID to fetch messages for
}
```

**Returns:**
```typescript
{
  success: true;
  messages: Array<{
    id: string;
    from_agent: string;
    to_agent: string | null;
    content: string;
    thread_id: string | null;
    created_at: number; // Unix epoch (seconds)
  }>;
}
```

## Tasks

### create_task

Create a new issue/task.

**Parameters:**
```typescript
{
  agent_id: string;        // creating agent ID
  title: string;           // task title
  description?: string;    // detailed description with clear acceptance criteria
  assignee?: string;       // assigned agent ID (optional)
  priority: "critical" | "high" | "medium" | "low";
}
```

**Returns:**
```typescript
{
  success: true;
  task_id: string;
}
```

---

### update_task

Change a task's status or assignee.

**Parameters:**
```typescript
{
  agent_id: string;    // calling agent ID
  task_id: string;     // task ID
  status?: "todo" | "in_progress" | "in_review" | "done";
  assignee?: string;   // new assignee (optional)
}
```

---

### get_my_tasks

Fetch tasks assigned to the calling agent.

**Parameters:**
```typescript
{
  agent_id: string; // agent ID to fetch tasks for
}
```

**Returns:**
```typescript
{
  success: true;
  tasks: Array<{
    id: string;
    title: string;
    description: string | null;
    status: string;
    priority: string;
    assignee: string | null;
    created_at: string;
  }>;
}
```

---

### claim_task

Atomically claim a task. Succeeds only if the task is in `todo` state and is assigned to (or unassigned from) the calling agent.
Prevents race conditions where two agents try to pick up the same task simultaneously.

**Parameters:**
```typescript
{
  agent_id: string; // agent ID claiming the task
  task_id: string;  // task ID to claim
}
```

**Returns:**
```typescript
{
  success: true;
  claimed: boolean;
  reason?: string; // reason if claimed: false
}
```

---

### get_all_tasks

Returns all tasks in the session regardless of assignee.

**Parameters:**
```typescript
{
  agent_id: string; // calling agent ID
}
```

**Returns:**
```typescript
{
  success: true;
  tasks: Array<{ id: string; title: string; status: string; assignee: string | null; priority: string; }>;
}
```

---

### get_team_status

Aggregates task counts by status for the entire session.
Agents use `has_pending_work` to decide when to broadcast `end:waiting` or `end:_done`.

**Parameters:**
```typescript
{
  agent_id: string; // calling agent ID
}
```

**Returns:**
```typescript
{
  success: true;
  todo: number;
  in_progress: number;
  in_review: number;
  done: number;
  has_pending_work: boolean; // todo + in_progress + in_review > 0
}
```

## Artifacts

### post_artifact

Share a deliverable (design spec, PR, report, etc.).

**Parameters:**
```typescript
{
  agent_id: string;  // sharing agent ID
  name: string;      // artifact name (e.g. "login-fe-pr")
  type: "figma_spec" | "pr" | "report" | "analytics_plan" | "design";
  content: string;   // artifact content (text or JSON string)
  task_id?: string;  // associated task ID (optional)
}
```

**Returns:**
```typescript
{
  success: true;
  artifact_id: string;
}
```

---

### get_artifact

Retrieve an artifact by name.

**Parameters:**
```typescript
{
  agent_id: string; // calling agent ID
  name: string;     // artifact name
}
```

**Returns:**
```typescript
{
  success: true;
  artifact: { id: string; name: string; type: string; content: string; created_by: string; task_id: string | null; };
}
// If not found: { success: false, artifact: null, error: "artifact not found" }
```

## Review

### request_review

Request a review for an artifact.

**Parameters:**
```typescript
{
  agent_id: string;    // requesting agent ID
  artifact_id: string; // artifact ID to review
  reviewer: string;    // reviewer agent ID (e.g. fe2, be2)
}
```

**Returns:**
```typescript
{
  success: true;
  review_id: string;
}
```

---

### submit_review

Submit a review result. Returns permission denied if the calling agent is not the designated reviewer.

**Parameters:**
```typescript
{
  agent_id: string;            // reviewer agent ID
  review_id: string;           // review ID
  status: "approved" | "changes_requested";
  comments?: string;           // review comments
}
```

## Memory

### read_memory

Read an agent's memory (or project memory).

**Parameters:**
```typescript
{
  agent_id?: string; // if omitted, returns combined project.md + lessons.md
}
```

**Returns:**
```typescript
{
  success: true;
  content: string | null; // markdown text. null if the file does not exist
}
```

---

### write_memory

Update a specific section of an agent's memory. Adds the section if it doesn't exist; replaces it if it does.
Writes to `project.md` when `agent_id` is omitted.

**Parameters:**
```typescript
{
  agent_id?: string; // agent ID. writes to project.md if omitted
  key: string;       // section key (e.g. "tech-stack", "conventions")
  content: string;   // new content
}
```

---

### append_memory

Append content with a datestamp to an agent's memory.
Appends to the shared `lessons.md` when `agent_id` is omitted.

**Parameters:**
```typescript
{
  agent_id?: string; // agent ID. appends to lessons.md if omitted
  content: string;   // content to append
}
```

## Sessions

### save_session_summary

At session end, saves the summary, tasks, and messages to `.relay/sessions/{session_id}/`.

**Parameters:**
```typescript
{
  agent_id: string;         // calling agent ID (usually the coordinator)
  session_id: string;       // session ID (YYYY-MM-DD-NNN format recommended)
  summary: string;          // session summary text
  tasks: object[];          // all tasks in the session
  messages: object[];       // all messages in the session
}
```

---

### list_sessions

**Parameters:**
```typescript
{
  agent_id: string; // calling agent ID
}
```

**Returns:**
```typescript
{
  success: true;
  sessions: string[]; // session ID list (newest first)
}
```

---

### get_session_summary

**Parameters:**
```typescript
{
  agent_id: string;   // calling agent ID
  session_id: string; // session ID
}
```

**Returns:**
```typescript
{
  success: true;
  summary: string; // contents of summary.md
}
```

## Visibility

### broadcast_thinking

Push an agent's current intent to the dashboard as an `agent:thinking` WebSocket event.
Fire-and-forget — does not write to the database.
Add this to your agent's `tools` list and call it before each significant operation.

**Parameters:**
```typescript
{
  agent_id: string; // calling agent ID
  content: string;  // what the agent is about to do or thinking about
}
```

**Returns:**
```typescript
{
  success: true;
}
```

:::tip
Call `broadcast_thinking` before each significant operation to make the agent's intent visible in the dashboard Thoughts panel.
:::
