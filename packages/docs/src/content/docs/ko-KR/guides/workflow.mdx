---

`agents.yml`의 `workflow` 섹션을 사용하면 에이전트가 작업하는 순서를 완전히 제어할 수 있습니다.

## 기본 구조

```yaml
workflow:
  jobs:
    {jobId}:
      agents: [에이전트 ID 목록]
      description: "이 job에서 수행하는 작업 설명"
      end:
        {nextJobId}: "이 조건이 충족되면 다음 job으로 진행"
        {anotherJobId}: "이 다른 조건이 충족되면 다른 job으로 진행"
```

## 특수 값

- `_done`: 워크플로우 종료를 나타내는 예약된 job ID
- `end` 조건이 여러 개 정의된 경우, 에이전트가 상황에 따라 선택합니다

## 기본 워크플로우 예시

```yaml
workflow:
  jobs:
    planning:
      agents: [pm]
      description: "사용자 요구사항을 분석하고 태스크로 분해합니다."
      end:
        design: "모든 태스크가 생성되고 배정되면"

    design:
      agents: [designer, da]
      description: "Designer는 UX 스펙을, DA는 분석 계획을 작성합니다."
      end:
        development: "두 산출물이 모두 post_artifact로 공유되면"

    development:
      agents: [fe, be]
      description: "FE는 UI를, BE는 API를 구현합니다. 둘 다 PR 아티팩트를 생성합니다."
      end:
        review: "구현이 완료되고 PR이 준비되면"

    review:
      agents: [fe2, be2]
      reviewers:
        fe: [fe2]
        be: [be2]
      description: "PR을 리뷰합니다."
      end:
        qa: "모든 PR이 LGTM을 받으면"
        development: "변경 요청이 있으면"

    qa:
      agents: [qa]
      description: "기능을 테스트하고 발견된 버그는 티켓으로 생성합니다."
      end:
        deploy: "모든 테스트가 통과되면"
        development: "버그가 발견되면"

    deploy:
      agents: [deployer]
      description: "QA를 통과한 기능을 프로덕션에 배포합니다."
      end:
        _done: "배포가 완료되면"
```

## reviewers 필드

리뷰가 필요한 job에서 `reviewers` 필드를 사용하세요:

```yaml
review:
  agents: [fe2, be2]
  reviewers:
    fe: [fe2]    # fe2가 fe가 생성한 아티팩트를 리뷰
    be: [be2]    # be2가 be가 생성한 아티팩트를 리뷰
```

리뷰어 에이전트는 해당 원작자(fe, be)의 페르소나로 실행되지만,
`agent_id`는 `fe2` 또는 `be2`로 설정됩니다.

## 간소화된 워크플로우

빠른 프로토타입이나 간단한 태스크에는 워크플로우를 단순하게 구성할 수 있어요:

```yaml
workflow:
  jobs:
    planning:
      agents: [pm]
      description: "태스크 분해"
      end:
        development: "태스크가 생성되면"

    development:
      agents: [fe, be]
      description: "구현"
      end:
        _done: "구현이 완료되면"
```

## 병렬 실행

`agents` 배열에 여러 에이전트를 나열하면 모두 동시에 실행됩니다:

```yaml
design:
  agents: [designer, da]  # 두 에이전트가 동시에 작업
```

배열의 모든 에이전트가 `end`를 선언해야 다음 job으로 넘어갑니다.
