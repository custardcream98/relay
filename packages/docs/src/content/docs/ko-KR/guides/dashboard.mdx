relay MCP 서버는 웹 대시보드도 함께 제공합니다.
에이전트들이 생각하고 서로 소통하는 과정을 실시간으로 확인할 수 있어요.

## 접속

MCP 서버가 시작되면 대시보드가 자동으로 실행됩니다:

```
http://localhost:3456
```

## 세 가지 패널

```
┌──────────────────────────────────────────────────────────┐
│  [PM ●]  [Designer ○]  [DA ○]  [FE ●]  [BE ●]  [QA ○]   │
├───────────────────┬──────────────────┬───────────────────┤
│    태스크 보드     │    메시지 피드    │    에이전트 생각   │
│    (Kanban)       │   (Slack 스타일)  │   (실시간 스트림)  │
└───────────────────┴──────────────────┴───────────────────┘
```

### 태스크 보드 (Kanban)

- 전체 이슈 상태를 Kanban 보드로 표시
- 태스크 상태 변경 시 즉시 업데이트: `todo` → `in_progress` → `in_review` → `done`
- 태스크 클릭 시 세부 정보 확인 (담당자, 우선순위, 설명)

### 메시지 피드

- 에이전트 간 대화를 Slack 스타일의 스레드로 표시
- `to: null` 브로드캐스트 메시지와 다이렉트 메시지를 구분
- 새 메시지가 실시간으로 표시됨

### 에이전트 생각

- 선택한 에이전트의 추론 과정(도구 호출 전 텍스트)을 실시간으로 스트리밍
- 에이전트 상태 표시기: `idle` (회색), `working` (초록), `waiting` (노랑)

## WebSocket 이벤트

대시보드는 WebSocket을 통해 실시간 업데이트를 받습니다.
모든 이벤트는 `{ type, timestamp, ...payload }` 구조를 따르며, 별도의 `payload` 래퍼 필드는 없습니다.

| 이벤트 타입 | 설명 |
|---|---|
| `agent:thinking` | 에이전트 추론 텍스트 스트림 |
| `agent:status` | 에이전트 상태 변경 (idle/working/waiting) |
| `message:new` | 새 메시지 수신 |
| `task:updated` | 태스크 상태 변경 |
| `artifact:posted` | 새 아티팩트 공유 |
| `review:requested` | 리뷰 요청 발생 |
| `memory:updated` | 에이전트 메모리 업데이트 |

## 히스토리 재생

모든 이벤트는 타임스탬프와 함께 SQLite에 저장됩니다.

세션이 종료된 후에도 대시보드에서 세션을 선택해 전체 과정을 다시 재생할 수 있어요:
- 에이전트들이 작업한 순서
- 어떤 메시지가 교환되었는지
- 어디서 블로커가 발생했는지

## REST API

대시보드는 Hono 기반 REST API도 제공합니다:

```
GET  /api/session               # 현재 세션 스냅샷 (태스크, 메시지, 아티팩트)
GET  /api/sessions/:id          # 특정 세션 요약 (summary.md)
GET  /api/sessions/:id/events   # 세션 이벤트 목록 (히스토리 재생용)
GET  /api/agents                # 에이전트 목록
POST /api/hook/tool-use         # PostToolUse 훅 수신기 (내부용)
```
