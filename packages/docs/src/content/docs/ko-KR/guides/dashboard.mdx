relay MCP 서버는 웹 대시보드도 함께 제공합니다.
에이전트들이 생각하고 서로 소통하는 과정을 실시간으로 확인할 수 있어요.

## 접속

MCP 서버가 시작되면 대시보드가 자동으로 실행됩니다.
기본 포트는 `3456`이지만, 이미 사용 중이면 `3456–3465` 범위에서 자동 선택됩니다.

실제 대시보드 URL은 `get_server_info`로 확인할 수 있습니다:

```typescript
get_server_info({ agent_id: "orchestrator" })
// → { dashboardUrl: "http://localhost:3456", port: 3456, ... }
```

```
http://localhost:3456   # 기본값 — 자동 선택된 경우 다를 수 있음
```

## 레이아웃

![relay 대시보드 — 라이브 세션](/relay/screenshots/dashboard-ko.png)

### Agent Arena (좌측 사이드바)

- 세션의 모든 에이전트를 현재 상태와 함께 표시: `idle` (회색), `working` (초록), `waiting` (노랑)
- 각 에이전트의 활성 태스크 수와 현재 생각 중인 내용 스니펫을 실시간으로 보여줌
- 에이전트를 클릭하면 Activity 피드가 해당 에이전트 이벤트에 포커스됨

### Activity Feed (메인 영역)

- 모든 세션 이벤트의 통합 실시간 타임라인: 메시지, 에이전트 추론 스트림, 태스크 변경, 아티팩트, 리뷰
- 이벤트 타입 또는 에이전트로 필터링 가능
- 추론 스트림 항목은 에이전트가 작업하는 동안 실시간 추론 텍스트를 표시

### Task Board (하단, 접기 가능)

- 모든 태스크의 Kanban 뷰: `todo` → `in_progress` → `in_review` → `done`
- 태스크 상태 변경 시 즉시 업데이트
- 태스크 클릭 시 세부 정보 확인 (담당자, 우선순위, 설명)

## WebSocket 이벤트

대시보드는 WebSocket을 통해 실시간 업데이트를 받습니다.
모든 이벤트는 `{ type, timestamp, ...payload }` 구조를 따르며, 별도의 `payload` 래퍼 필드는 없습니다.

| 이벤트 타입 | 설명 |
|---|---|
| `agent:thinking` | 에이전트 추론 텍스트 스트림 |
| `agent:status` | 에이전트 상태 변경 (idle/working/waiting) |
| `message:new` | 새 메시지 수신 |
| `task:updated` | 태스크 상태 변경 |
| `artifact:posted` | 새 아티팩트 공유 |
| `review:requested` | 리뷰 요청 발생 |
| `review:updated` | 리뷰 결과 제출 (approved/changes_requested) |
| `memory:updated` | 에이전트 메모리 업데이트 |
| `session:started` | 새 세션 활성화 (대시보드 초기화) |
| `session:snapshot` | WebSocket 연결 시 전체 세션 상태 전송 (초기 로딩용) |

## REST API

대시보드는 Hono 기반 REST API도 제공합니다:

```
GET  /api/session              # 현재 세션 스냅샷 (태스크, 메시지, 아티팩트)
GET  /api/sessions/:id         # 세션 요약 (save_session_summary가 저장한 summary.md)
GET  /api/agents               # 에이전트 목록 (풀 파일에서)
POST /api/hook/tool-use        # PostToolUse 훅 수신기 (내부용)
```
