A guide for applying relay to a project for the first time after installation.

<Steps>

1. **Scan the project (once per project)**

   Run this the first time you use relay in a new project, or when the team needs to re-learn the codebase:

   ```
   /relay:init
   ```

   All configured agents spawn in parallel, scan the codebase from their own perspective,
   and save project memory to `.relay/memory/`.

2. **Run a task**

   Once init is complete, you can request tasks immediately:

   ```
   /relay:relay "Add a shopping cart feature"
   ```

   Agents collaborate event-driven according to the configured workflow.

3. **Check the dashboard (optional)**

   Once the MCP server is running, you can monitor agent activity in real time on the web dashboard:

   ```
   http://localhost:3456
   ```

4. **Invoke a single agent (when needed)**

   When you only need one specific agent's expertise:

   ```
   /relay:agent fe "Refactor the CartItem component"
   /relay:agent be "Security review the user auth API"
   ```

</Steps>

## Example flow

```
User: /relay:relay "Implement a login feature"

[PM]       → breaks down login into 5 tasks
             → FE: UI, BE: API, Designer: screen spec, DA: event design

[Designer] → writes login screen spec (post_artifact)
[DA]       → designs login_success, login_fail events (post_artifact)

[FE]       → implements login UI, creates fe-pr artifact
[BE]       → implements auth API, creates be-pr artifact

[FE2]      → reviews FE PR (request_review → submit_review)
[BE2]      → reviews BE PR

[QA]       → writes test scenarios, validates
           → creates tickets for any bugs found → dev team fixes

[Deployer] → deployment complete, notifies team

User: ✅ Login feature deployed!
```
