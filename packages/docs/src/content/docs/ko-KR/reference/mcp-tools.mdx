title: MCP 도구 레퍼런스
description: relay MCP 서버가 제공하는 모든 18개 도구의 파라미터와 반환값
---

모든 도구는 성공 시 `{ success: boolean, ...fields }` 형태의 flat 객체로, 실패 시 `{ success: false, error: string }`으로 응답합니다 — 중첩된 `data` 필드는 없습니다.
`agent_id`는 호출자를 추적하기 위해 항상 파라미터에 포함됩니다.
`session_id`는 서버가 `RELAY_SESSION_ID` 환경 변수에서 자동으로 주입하므로, 클라이언트가 직접 전달할 필요가 없습니다.

## 메시징

### send_message

에이전트 간 메시지를 전송합니다.

**파라미터:**
```typescript
{
  agent_id: string;    // 발신 에이전트 ID
  to: string | null;   // 수신 에이전트 ID. null = 브로드캐스트
  content: string;     // 메시지 내용
  thread_id?: string;  // 스레드 ID (선택사항)
}
```

**반환값:**
```typescript
{
  success: true;
  message_id: string;
}
```

---

### get_messages

수신된 메시지를 가져옵니다 (다이렉트 메시지 + 브로드캐스트).

**파라미터:**
```typescript
{
  agent_id: string; // 메시지를 가져올 에이전트 ID
}
```

**반환값:**
```typescript
{
  success: true;
  messages: Array<{
    id: string;
    from_agent: string;
    to_agent: string | null;
    content: string;
    thread_id: string | null;
    created_at: number; // Unix epoch (초)
  }>;
}
```

## 태스크

### create_task

새 이슈/태스크를 생성합니다.

**파라미터:**
```typescript
{
  agent_id: string;        // 생성 에이전트 ID
  title: string;           // 태스크 제목
  description?: string;    // 명확한 완료 조건이 포함된 상세 설명
  assignee?: string;       // 담당 에이전트 ID (선택사항)
  priority: "critical" | "high" | "medium" | "low";
}
```

**반환값:**
```typescript
{
  success: true;
  task_id: string;
}
```

---

### update_task

태스크의 상태나 담당자를 변경합니다.

**파라미터:**
```typescript
{
  agent_id: string;    // 호출 에이전트 ID
  task_id: string;     // 태스크 ID
  status?: "todo" | "in_progress" | "in_review" | "done";
  assignee?: string;   // 새 담당자 (선택사항)
}
```

---

### get_my_tasks

호출 에이전트에게 배정된 태스크를 가져옵니다.

**파라미터:**
```typescript
{
  agent_id: string; // 태스크를 가져올 에이전트 ID
}
```

**반환값:**
```typescript
{
  success: true;
  tasks: Array<{
    id: string;
    title: string;
    description: string | null;
    status: string;
    priority: string;
    assignee: string | null;
    created_at: string;
  }>;
}
```

---

### claim_task

태스크를 원자적으로 클레임합니다. 태스크가 `todo` 상태이고 호출 에이전트에게 배정되어 있거나 미배정 상태일 때만 성공합니다.
두 에이전트가 동시에 같은 태스크를 클레임하려는 레이스 컨디션을 방지합니다.

**파라미터:**
```typescript
{
  agent_id: string; // 클레임하는 에이전트 ID
  task_id: string;  // 클레임할 태스크 ID
}
```

**반환값:**
```typescript
{
  success: true;
  claimed: boolean;
  reason?: string; // claimed: false인 경우의 이유
}
```

---

### get_all_tasks

세션의 모든 태스크를 담당자에 상관없이 반환합니다.

**파라미터:**
```typescript
{
  agent_id: string; // 호출 에이전트 ID
}
```

**반환값:**
```typescript
{
  success: true;
  tasks: Array<{ id: string; title: string; status: string; assignee: string | null; priority: string; }>;
}
```

---

### get_team_status

세션 전체의 태스크 수를 상태별로 집계합니다.
에이전트가 `end:waiting` 또는 `end:_done`을 브로드캐스트할 시점을 결정하는 데 `has_pending_work`를 사용합니다.

**파라미터:**
```typescript
{
  agent_id: string; // 호출 에이전트 ID
}
```

**반환값:**
```typescript
{
  success: true;
  todo: number;
  in_progress: number;
  in_review: number;
  done: number;
  has_pending_work: boolean; // todo + in_progress + in_review > 0
}
```

## 아티팩트

### post_artifact

산출물(디자인 스펙, PR, 리포트 등)을 공유합니다.

**파라미터:**
```typescript
{
  agent_id: string;  // 공유하는 에이전트 ID
  name: string;      // 아티팩트 이름 (예: "login-fe-pr")
  type: "figma_spec" | "pr" | "report" | "analytics_plan" | "design" | "document";
  content: string;   // 아티팩트 내용 (텍스트 또는 JSON 문자열)
  task_id?: string;  // 연관된 태스크 ID (선택사항)
}
```

**반환값:**
```typescript
{
  success: true;
  artifact_id: string;
}
```

---

### get_artifact

이름으로 아티팩트를 가져옵니다.

**파라미터:**
```typescript
{
  agent_id: string; // 호출 에이전트 ID
  name: string;     // 아티팩트 이름
}
```

**반환값:**
```typescript
{
  success: true;
  artifact: { id: string; name: string; type: string; content: string; created_by: string; task_id: string | null; };
}
// 찾지 못한 경우: { success: false, artifact: null, error: "artifact not found" }
```

## 리뷰

### request_review

아티팩트에 대한 리뷰를 요청합니다.

**파라미터:**
```typescript

  agent_id: string;    // 요청 에이전트 ID

}
```

**반환값:**
```typescript
{
  success: true;
  review_id: string;
}
```

---

### submit_review

리뷰 결과를 제출합니다. 호출 에이전트가 지정된 리뷰어가 아닌 경우 권한 오류를 반환합니다.

**파라미터:**
```typescript
{
  agent_id: string;            // 리뷰어 에이전트 ID
  review_id: string;           // 리뷰 ID
  status: "approved" | "changes_requested";
  comments?: string;           // 리뷰 코멘트
}
```

## 메모리

### read_memory

에이전트의 메모리(또는 프로젝트 메모리)를 읽습니다.

**파라미터:**
```typescript
{
  agent_id?: string; // 생략하면 project.md만 반환
}
```

**반환값:**
```typescript
{
  success: true;
  content: string | null; // 마크다운 텍스트. 파일이 없으면 null
}
```

---

### write_memory

에이전트 메모리의 특정 섹션을 업데이트합니다. 섹션이 없으면 추가하고, 있으면 교체합니다.
`agent_id`를 생략하면 `project.md`에 씁니다.

**파라미터:**
```typescript
{
  agent_id?: string; // 에이전트 ID. 생략하면 project.md에 씀
  key: string;       // 섹션 키 (예: "tech-stack", "conventions")
  content: string;   // 새 내용
}
```

---

### append_memory

에이전트 개인 메모리 파일에 날짜스탬프와 함께 내용을 추가합니다.
`agent_id`는 필수입니다 — 세션 회고는 `save_session_summary`를 사용하세요.

**파라미터:**
```typescript
{
  agent_id: string; // 에이전트 ID (필수)
  content: string;  // 추가할 내용
}
```

## 세션

### save_session_summary

세션 종료 시 요약을 `.relay/sessions/{session_id}/summary.md`에 저장합니다.
태스크와 메시지는 이미 SQLite에 저장되어 있으므로 별도로 전달할 필요가 없습니다.

**파라미터:**
```typescript
{
  agent_id: string;         // 호출 에이전트 ID (보통 코디네이터)
  session_id: string;       // 세션 ID (YYYY-MM-DD-NNN-XXXX 형식)
  summary: string;          // 세션 요약 텍스트
}
```

---

### list_sessions

과거 세션 ID를 역순(최신순)으로 가져옵니다.

**파라미터:**
```typescript
{
  agent_id: string; // 호출 에이전트 ID
}
```

**반환값:**
```typescript
{
  success: true;
  sessions: string[]; // 세션 ID 목록 (최신순)

```

---

### get_session_summary

특정 세션의 요약을 가져옵니다.

**파라미터:**
```typescript
{
  agent_id: string;   // 호출 에이전트 ID

}
```

**반환값:**
```typescript
{
  success: true;
  summary: string; // summary.md 내용
}
```

## 에이전트

### list_agents

주어진 세션의 에이전트 목록을 가져옵니다 (`.relay/session-agents-{session_id}.yml`이 있으면 해당 파일에서, 없으면 풀 파일에서 로드).

**파라미터:**
```typescript
{
  agent_id: string;      // 호출 에이전트 ID
  session_id?: string;   // 세션 ID — 지정 시 /relay:relay Team Composition 단계에서
                         // 작성된 세션별 팀 파일을 로드합니다
}
```

**반환값:**
```typescript
{
  success: true;
  agents: Array<{
    id: string;
    name: string;
    emoji: string;
    description: string;
    tools: string[];
    systemPrompt: string;
  }>;
}
```

---

### get_workflow

로드된 세션 설정에서 `workflow` 구성을 가져옵니다.

**파라미터:**
```typescript
{
  agent_id: string; // 호출 에이전트 ID
}
```

**반환값:**
```typescript
{
  jobs: Record<string, {
    agents: string[];
    description: string;
    reviewers?: Record<string, string[]>;
    end: Record<string, string>;
  }>;
}
```

## 가시성

### broadcast_thinking

에이전트의 현재 의도를 `agent:thinking` WebSocket 이벤트로 대시보드에 실시간 전송합니다.
DB에 기록하지 않는 fire-and-forget 방식입니다.
에이전트의 `tools` 목록에 추가하고, 중요한 작업 전에 호출하세요.

**파라미터:**
```typescript
{
  agent_id: string; // 호출 에이전트 ID
  content: string;  // 에이전트가 하려는 작업 또는 현재 생각
}
```

**반환값:**
```typescript
{
  success: true;
}
```

:::tip
각 주요 작업 전에 `broadcast_thinking`을 호출하면 대시보드 Thoughts 패널에 에이전트의 의도가 실시간으로 표시됩니다.
:::
