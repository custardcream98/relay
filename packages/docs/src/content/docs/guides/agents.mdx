Agent personas in relay are fully customisable via a YAML file.

## File structure

```
relay/
├── agents.default.yml   # default personas (do not modify)
└── agents.yml           # your customisations (edit this file)
```

Do not modify `agents.default.yml`. All customisations go in `agents.yml`.
`agents.yml` overrides `agents.default.yml`.

## Basic structure of agents.yml

```yaml
agents:
  fe:
    # only specify fields you want to override
    systemPrompt: |
      You are a React and TypeScript expert.
      ...
```

## Override

Change only certain fields of an existing agent:

```yaml
agents:
  fe:
    name: "Frontend Engineer (React specialist)"
    systemPrompt: |
      You are a Frontend Engineer at a startup.
      Your primary stack is React, TypeScript, and Tailwind CSS.
      ...
```

## extends

Inherit and extend another agent's configuration:

```yaml
agents:
  fe-mobile:
    extends: fe          # inherit from the fe persona
    name: "Mobile Frontend Engineer"
    systemPrompt: |
      You are a Frontend Engineer specialising in React Native.
      ...
```

## disabled

Disable a specific agent:

```yaml
agents:
  da:
    disabled: true       # disable the DA agent
```

Disabled agents are excluded from `list_agents` results and from all workflows.

## Adding custom agents

You can add entirely new personas beyond the defaults:

```yaml
agents:
  security:
    name: "Security Engineer"
    emoji: "🔐"
    description: "Security vulnerability analysis and remediation"
    tools:
      - get_messages
      - get_my_tasks
      - update_task
      - get_artifact
      - post_artifact
      - send_message
    systemPrompt: |
      You are a security expert.
      Review code against the OWASP Top 10 and
      identify vulnerabilities with recommended fixes.
```

## tools field

Specifies the list of MCP tools the agent is allowed to call:

```yaml
tools:
  - send_message
  - get_messages
  - create_task
  - update_task
  - get_my_tasks
  - post_artifact
  - get_artifact
  - request_review
  - submit_review
  - read_memory
  - write_memory
  - append_memory
```

An agent cannot call any MCP tool not listed in its `tools` array.
It is recommended to grant only the minimum tools necessary for each role.
