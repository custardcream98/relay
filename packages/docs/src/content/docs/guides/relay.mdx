`/relay:relay` takes a user task and has the entire team defined in `agents.yml` collaborate to handle it.
All agents spawn simultaneously at session start and work event-driven.

## Usage

```
/relay:relay "description of what to implement"
```

Examples:
```
/relay:relay "Add a shopping cart feature"
/relay:relay "Improve the user profile page"
/relay:relay "Harden the payment API security"
```

## Pre-flight checks

Before running, the skill verifies:
- relay MCP server connection (via `list_agents`)
  - If no agents are found, guides the user to create `agents.yml` first
- Whether `.relay/memory/project.md` exists (suggests `/relay:init` if not)
- Prints dashboard URL: `http://localhost:3456`

## Session start

Loads the full team via `list_agents` and **spawns all agents simultaneously**.

For each agent spawn:
1. Load persona from `list_agents`
2. Load memory via `read_memory(agent_id)` + `read_memory()` (project.md + lessons.md)
3. Restrict MCP tool access to the agent's `tools` array

The user's request is added to the coordinator agent's system prompt.
Other agents wait for the coordinator to assign tasks.

## Event-driven collaboration model

Agents do not communicate directly with each other. All collaboration goes through MCP tools:

- `send_message` / `get_messages` тАФ inter-agent messaging
- `create_task` / `claim_task` тАФ task creation and atomic claiming
- `post_artifact` / `get_artifact` тАФ sharing deliverables
- `request_review` / `submit_review` тАФ code review

When an agent has no more work or has finished, it broadcasts an end declaration:

```
"end:waiting | {reason}"   # no current work (session continues)
"end:_done | {summary}"    # fully complete
"end:failed | {reason}"    # unrecoverable error
```

The orchestrator re-spawns a `end:waiting` agent when new tasks are assigned to it.

## Reviewer spawn

When an agent broadcasts `"Review requested: {reviewerId}"`,
the orchestrator immediately spawns that reviewer:

- If `reviewerId` exists in `list_agents`, uses that persona
- Otherwise, inherits the requester's persona

## Session end

When all agents have declared `end:_done`:
1. `append_memory(content: ...)` тАФ save team retrospective to `lessons.md`
2. `save_session_summary(session_id, summary, tasks, messages)` тАФ archive the session
3. Report the results summary to the user
