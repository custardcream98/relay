프로젝트에 relay를 처음 도입할 때 실행하세요. 설정된 에이전트 전원이 동시에 코드베이스를 읽고, 각자의 관점에서 파악한 내용을 `.relay/memory/`에 기록합니다. 이후 모든 세션은 이 메모리를 기반으로 시작합니다.

## 언제 실행하나요?

- **최초 1회** — 프로젝트에서 relay를 처음 쓸 때
- **재실행** — 코드베이스 구조가 크게 바뀌었을 때

`.relay/memory/`가 없는 상태에서 `/relay:relay`를 실행하면 자동으로 먼저 실행하도록 안내합니다.

## 실행 흐름

### 에이전트 풀이 없을 때

`.relay/agents.pool.yml`이 아직 없으면, relay가 먼저 프로젝트를 분석해서 도메인에 맞는 팀을 제안합니다.

```
[1] 이 풀로 .relay/agents.pool.yml 생성 (권장)
[2] 예시 풀 사용 (agents.pool.example.yml)
[3] agents.pool.yml을 직접 작성할게요
```

풀 파일이 생성되면 자동으로 스캔을 이어서 진행합니다.

### 병렬 스캔

풀에 정의된 **에이전트 전원이 동시에** 코드베이스를 탐색합니다. 각 에이전트는 자신의 역할에 맞는 부분을 파악합니다 — FE는 컴포넌트 구조를, BE는 API 패턴을, PM은 전체 아키텍처를.

탐색이 끝난 에이전트는 각자의 메모리 파일에 주요 발견 사항을 기록하고, 한 에이전트가 전체를 종합해 `project.md`에 요약을 작성합니다.

## 출력 결과

```
.relay/
└── memory/
    ├── project.md        ← 전체 요약 (모든 세션에 주입됨)
    └── agents/
        ├── pm.md
        ├── fe.md
        ├── be.md
        └── ...           ← 에이전트별 개인 메모리
```

<Aside type="tip">
`.relay/memory/`를 git에 커밋하세요. 팀 전원이 동일한 에이전트 컨텍스트를 공유합니다.
</Aside>

## 이후 단계

init이 완료되면 바로 작업을 시작할 수 있습니다:

```
/relay:relay "장바구니 기능 추가해줘"
```
