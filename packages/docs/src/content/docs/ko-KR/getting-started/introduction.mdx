relay는 Claude Code 서브에이전트들이 각자의 페르소나를 갖고 협업하는 **도메인에 구애받지 않는** 멀티에이전트 프레임워크입니다.
`agents.yml`에 원하는 팀을 자유롭게 정의할 수 있어요 — 웹 개발팀, 리서치팀, 마케팅팀, 법무팀, 무엇이든 가능합니다.

Claude Code의 Agent 도구만 사용하므로 추가 API 비용이 발생하지 않아요.

## 동작 방식

```
사용자: "장바구니 기능을 추가해줘"
  ↓
[PM]       → 태스크 분해, 이슈 생성
[Designer] → UX 흐름, 컴포넌트 스펙
[DA]       → 이벤트 설계, KPI 정의
[BE]       → API 설계 및 구현
[FE]       → UI 구현
[FE2]      → FE 코드 리뷰
[BE2]      → BE 코드 리뷰
[QA]       → 테스트 시나리오 및 버그 리포트
[Deployer] → 배포
```

## 세 가지 레이어

| 레이어 | 역할 |
|---|---|
| MCP server | 에이전트 간 통신 인프라 (메시지 버스, 태스크 보드, 메모리) |
| Skills | `/relay:relay`, `/relay:init`, `/relay:agent` 오케스트레이션 전략 |
| Hooks | PostToolUse — MCP 도구 호출 시 대시보드 실시간 업데이트 |

## 에이전트는 직접 정의해요

에이전트 설정은 `agents.yml`을 통해 완전히 커스터마이즈할 수 있습니다.
`agents.example.yml`에는 웹 개발팀 예시(pm, designer, da, fe, be, qa, deployer)가 포함되어 있어요.
그대로 복사해서 시작하거나, 처음부터 나만의 팀을 정의할 수 있습니다.

## relay를 쓰는 이유

대부분의 AI 에이전트 도구는 단일 에이전트로 모든 것을 처리합니다. relay는 다릅니다.

각 에이전트는 자신의 역할만 담당하며, MCP 서버를 통해 다른 에이전트와 직접 소통합니다.
피어 투 피어 방식으로 — 중앙 오케스트레이터 없이 이벤트 드리븐으로 협업합니다.
Claude Code의 Agent 도구만 사용하므로 추가 API 비용이 발생하지 않아요.
