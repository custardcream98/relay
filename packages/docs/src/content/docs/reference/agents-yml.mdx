`agents.yml` is the configuration file for customising agent personas and workflows.
It overrides the defaults in `agents.default.yml`.

## Top-level structure

```yaml
agents:
  {agentId}:
    # agent configuration
    ...

workflow:
  jobs:
    # workflow configuration
    ...
```

## agents section

### Agent schema

```yaml
agents:
  {agentId}:
    name: string           # display name (e.g. "Frontend Engineer")
    emoji: string          # emoji icon (e.g. "💻")
    description: string    # short role description
    disabled: boolean      # disable the agent when true (default: false)
    extends: string        # agent ID to inherit from
    tools:                 # list of allowed MCP tools
      - tool_name
    systemPrompt: string   # agent system prompt (multiline)
```

### Default agent IDs

| ID | Name |
|---|---|
| `pm` | Product Manager |
| `designer` | UX Designer |
| `da` | Data Analyst |
| `fe` | Frontend Engineer |
| `be` | Backend Engineer |
| `qa` | QA Engineer |
| `deployer` | Deployer |

## workflow section

### job schema

```yaml
workflow:
  jobs:
    {jobId}:
      agents:              # list of agent IDs to run in parallel in this job
        - agentId
      description: string  # job description (injected into agent system prompts)
      reviewers:           # reviewer mapping (optional)
        {agentId}: [reviewerAgentId]
      end:                 # conditions for choosing the next job
        {nextJobId}: "condition description"
        _done: "workflow end condition"
```

### Special job IDs

- `_done`: reserved keyword that marks the end of the workflow

### Start job detection

The start job is determined automatically: the job that does not appear as a destination in any other job's `end` map is the entry point.

## Full example

```yaml
agents:
  fe:
    name: "Frontend Engineer (Next.js)"
    systemPrompt: |
      You are a Frontend Engineer specialising in Next.js App Router.
      Your primary stack is TypeScript, Tailwind CSS, and React Query.

      ## Role
      - Implement Next.js components from design specs
      - Create PR artifacts and request code review from FE2

      ## Rules
      - Set agent_id to "fe" normally, or "fe2" when acting as a reviewer

  security:
    name: "Security Engineer"
    emoji: "🔐"
    description: "Security vulnerability analysis"
    tools:
      - get_messages
      - get_artifact
      - post_artifact
      - send_message
    systemPrompt: |
      You are a security expert.
      Review PRs against the OWASP Top 10.

  da:
    disabled: true  # disable the DA agent

workflow:
  jobs:
    planning:
      agents: [pm]
      description: "Analyse requirements and break down into tasks"
      end:
        development: "When tasks are assigned"

    development:
      agents: [fe, be]
      description: "Implement"
      end:
        security-review: "When implementation is complete"

    security-review:
      agents: [security]
      description: "Security review"
      end:
        qa: "When no security issues are found"
        development: "When fixes are needed"

    qa:
      agents: [qa]
      description: "Feature testing"
      end:
        deploy: "When tests pass"
        development: "When bugs are found"

    deploy:
      agents: [deployer]
      description: "Deploy"
      end:
        _done: "When deployment is complete"
```
