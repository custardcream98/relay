`/relay:agent`는 특정 에이전트 하나를 독립적으로 호출합니다.
전체 팀 워크플로우 대신 특정 에이전트의 전문성만 필요할 때 사용하세요.

## 사용법

```
/relay:agent {agentId} "{지시사항}"
```

예시:
```
/relay:agent fe "CartItem 컴포넌트를 리팩터링해줘"
/relay:agent be "사용자 인증 API 보안 검토해줘"
/relay:agent qa "로그인 기능의 엣지 케이스 테스트 시나리오를 작성해줘"
/relay:agent designer "체크아웃 플로우의 UX 개선안을 작성해줘"
```

## 사용 가능한 에이전트 ID

사용 가능한 에이전트 목록은 `agents.yml` 설정에 따라 달라집니다.
`list_agents` 결과에서 ID를 확인하세요.

## 실행 단계

1. `list_agents`로 사용 가능한 에이전트 확인
2. 대시보드 URL 출력: `http://localhost:3456`
3. 지정된 에이전트의 페르소나와 메모리 로드
4. `tools` 배열로 MCP 도구 접근을 제한해 에이전트를 독립적으로 실행
5. 완료 후 `append_memory`로 학습 내용 저장

## 메모리 로드

단일 에이전트도 전체 프로젝트 컨텍스트를 활용합니다:
1. `read_memory(agent_id: "{agentId}")` — 에이전트 개인 메모리
2. `read_memory()` — 프로젝트 메모리 (`project.md` + `lessons.md`)

두 결과를 합쳐 시스템 프롬프트 앞에 주입합니다.

## 알 수 없는 에이전트 처리

지정한 에이전트 ID가 `list_agents`에 없으면:
- 사용 가능한 에이전트 목록을 사용자에게 표시
- 다른 에이전트를 선택하도록 안내

## 언제 사용하나요

**단일 에이전트 호출이 적합한 경우:**
- 특정 파일의 코드 리뷰가 필요할 때
- 아키텍처 분석이나 기술적 조언이 필요할 때
- 전체 워크플로우 없이 특정 산출물(테스트, 문서)만 필요할 때
- 빠르게 처리할 수 있어 팀 전체를 실행할 필요가 없을 때
