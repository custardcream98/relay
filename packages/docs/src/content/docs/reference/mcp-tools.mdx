---
title: MCP 툴 레퍼런스
description: relay MCP 서버가 제공하는 모든 툴의 파라미터와 반환값
---

모든 툴은 `{ success: boolean, data?, error? }` 형태로 응답한다.
파라미터에 `agent_id`가 항상 포함되어 누가 호출하는지 추적한다.
`session_id`는 서버가 환경변수(`RELAY_SESSION_ID`)에서 자동으로 주입하므로 클라이언트가 전달할 필요가 없다.

## 메시징

### send_message

에이전트 간 메시지를 전송한다.

**파라미터:**
```typescript
{
  agent_id: string;    // 발신 에이전트 ID
  to: string | null;   // 수신 에이전트 ID. null이면 브로드캐스트
  content: string;     // 메시지 내용
  thread_id?: string;  // 스레드 ID (선택)
}
```

**반환:**
```typescript
{
  success: true;
  message_id: string;
}
```

---

### get_messages

수신된 메시지를 조회한다 (직접 수신 + 브로드캐스트 포함).

**파라미터:**
```typescript
{
  agent_id: string; // 조회할 에이전트 ID
}
```

**반환:**
```typescript
{
  success: true;
  messages: Array<{
    id: string;
    from_agent: string;
    to_agent: string | null;
    content: string;
    thread_id: string | null;
    created_at: number; // Unix epoch (초)
  }>;
}
```

## 태스크

### create_task

새 이슈/태스크를 생성한다.

**파라미터:**
```typescript
{
  agent_id: string;        // 생성 에이전트 ID
  title: string;           // 태스크 제목
  description?: string;    // 완료 기준이 명확한 상세 설명
  assignee?: string;       // 담당 에이전트 ID (선택)
  priority: "critical" | "high" | "medium" | "low";
}
```

**반환:**
```typescript
{
  success: true;
  task_id: string;
}
```

---

### update_task

태스크 상태 또는 담당자를 변경한다.

**파라미터:**
```typescript
{
  agent_id: string;    // 호출 에이전트 ID
  task_id: string;     // 태스크 ID
  status?: "todo" | "in_progress" | "in_review" | "done";
  assignee?: string;   // 담당자 변경 시
}
```

---

### get_my_tasks

내 담당 태스크를 조회한다.

**파라미터:**
```typescript
{
  agent_id: string; // 조회할 에이전트 ID
}
```

**반환:**
```typescript
{
  success: true;
  tasks: Array<{
    id: string;
    title: string;
    description: string | null;
    status: string;
    priority: string;
    assignee: string | null;
    created_at: string;
  }>;
}
```

---

### claim_task

태스크를 원자적으로 클레임한다. `todo` 상태이고 해당 에이전트에 할당되어 있거나 미할당인 경우에만 성공한다.
두 에이전트가 동일한 태스크를 동시에 가져가는 race condition을 방지한다.

**파라미터:**
```typescript
{
  agent_id: string; // 클레임할 에이전트 ID
  task_id: string;  // 클레임할 태스크 ID
}
```

**반환:**
```typescript
{
  success: true;
  claimed: boolean;
  reason?: string; // claimed: false인 경우 사유
}
```

---

### get_all_tasks

담당자와 무관하게 세션 내 모든 태스크를 반환한다.

**파라미터:**
```typescript
{
  agent_id: string; // 호출 에이전트 ID
}
```

**반환:**
```typescript
{
  success: true;
  tasks: Array<{ id: string; title: string; status: string; assignee: string | null; priority: string; }>;
}
```

---

### get_team_status

세션 전체 태스크를 상태별로 집계한다.
`has_pending_work`를 통해 에이전트가 `end:waiting` 또는 `end:_done`을 선언할 시점을 결정한다.

**파라미터:**
```typescript
{
  agent_id: string; // 호출 에이전트 ID
}
```

**반환:**
```typescript
{
  success: true;
  todo: number;
  in_progress: number;
  in_review: number;
  done: number;
  has_pending_work: boolean; // todo + in_progress + in_review > 0
}
```

## 아티팩트

### post_artifact

산출물(디자인 스펙, PR, 리포트 등)을 공유한다.

**파라미터:**
```typescript
{
  agent_id: string;  // 공유 에이전트 ID
  name: string;      // 아티팩트 이름 (예: "login-fe-pr")
  type: "figma_spec" | "pr" | "report" | "analytics_plan" | "design";
  content: string;   // 아티팩트 내용 (텍스트 또는 JSON 문자열)
  task_id?: string;  // 연관 태스크 ID (선택)
}
```

**반환:**
```typescript
{
  success: true;
  artifact_id: string;
}
```

---

### get_artifact

아티팩트를 이름으로 조회한다.

**파라미터:**
```typescript
{
  agent_id: string; // 조회 에이전트 ID
  name: string;     // 아티팩트 이름
}
```

**반환:**
```typescript
{
  success: true;
  artifact: { id: string; name: string; type: string; content: string; created_by: string; task_id: string | null; };
}
// 없으면: { success: false, artifact: null, error: "artifact not found" }
```

## 리뷰

### request_review

아티팩트에 대한 리뷰를 요청한다.

**파라미터:**
```typescript
{
  agent_id: string;    // 요청 에이전트 ID
  artifact_id: string; // 리뷰 대상 아티팩트 ID
  reviewer: string;    // 리뷰어 에이전트 ID (예: fe2, be2)
}
```

**반환:**
```typescript
{
  success: true;
  review_id: string;
}
```

---

### submit_review

리뷰 결과를 제출한다. 호출 에이전트가 지정된 리뷰어가 아닌 경우 permission denied 반환.

**파라미터:**
```typescript
{
  agent_id: string;            // 리뷰어 에이전트 ID
  review_id: string;           // 리뷰 ID
  status: "approved" | "changes_requested";
  comments?: string;           // 리뷰 코멘트
}
```

## 메모리

### read_memory

에이전트의 기억(또는 프로젝트 기억)을 읽는다.

**파라미터:**
```typescript
{
  agent_id?: string; // 생략하면 project.md + lessons.md 합성 반환
}
```

**반환:**
```typescript
{
  success: true;
  content: string | null; // 마크다운 텍스트. 파일이 없으면 null
}
```

---

### write_memory

에이전트 기억의 특정 섹션을 갱신한다. 섹션이 없으면 추가, 있으면 교체.
`agent_id` 생략 시 `project.md`에 쓴다.

**파라미터:**
```typescript
{
  agent_id?: string; // 에이전트 ID. 생략하면 project.md에 쓴다
  key: string;       // 섹션 키 (예: "tech-stack", "conventions")
  content: string;   // 새 내용
}
```

---

### append_memory

에이전트 기억에 내용을 날짜스탬프와 함께 누적 추가한다.
`agent_id` 생략 시 공유 `lessons.md`에 추가한다.

**파라미터:**
```typescript
{
  agent_id?: string; // 에이전트 ID. 생략하면 lessons.md에 추가
  content: string;   // 추가할 내용
}
```

## 세션

### save_session_summary

세션 종료 시 요약, 태스크, 메시지를 `.relay/sessions/{session_id}/`에 저장한다.

**파라미터:**
```typescript
{
  agent_id: string;         // 호출 에이전트 ID (보통 오케스트레이터)
  session_id: string;       // 세션 ID (YYYY-MM-DD-NNN 형식 권장)
  summary: string;          // 세션 요약 텍스트
  tasks: object[];          // 세션 내 모든 태스크
  messages: object[];       // 세션 내 모든 메시지
}
```

---

### list_sessions

과거 세션 ID 목록을 최신순으로 조회한다.

**파라미터:**
```typescript
{
  agent_id: string; // 호출 에이전트 ID
}
```

**반환:**
```typescript
{
  success: true;
  sessions: string[]; // 세션 ID 목록 (최신순)
}
```

---

### get_session_summary

특정 세션의 요약을 조회한다.

**파라미터:**
```typescript
{
  agent_id: string;   // 호출 에이전트 ID
  session_id: string; // 세션 ID
}
```

**반환:**
```typescript
{
  success: true;
  summary: string; // summary.md 내용
}
```

## 에이전트

### list_agents

`agents.yml`에 정의된 에이전트 목록을 조회한다.

**파라미터:**
```typescript
{
  agent_id: string; // 호출 에이전트 ID
}
```

**반환:**
```typescript
{
  success: true; // (배열 직접 반환)
  // 실제로는 배열이 JSON 텍스트로 반환됨
  agents: Array<{
    id: string;
    name: string;
    emoji: string;
    description: string;
    tools: string[];
    systemPrompt: string;
  }>;
}
```

---

### get_workflow

`agents.yml`의 `workflow` 설정을 조회한다.

**파라미터:**
```typescript
{
  agent_id: string; // 호출 에이전트 ID
}
```

**반환:**
```typescript
{
  jobs: Record<string, {
    agents: string[];
    description: string;
    reviewers?: Record<string, string[]>;
    end: Record<string, string>;
  }>;
}
```
